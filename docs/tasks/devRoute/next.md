# 下一步优先建议与 v1 范围

> 基于当前状态，建议把"短期体验质变"和"长期伴侣架构"分开推进：前者先提升存在感，后者补足关系、记忆、主动性和边界。

## v1 定位

v1 的目标不是做一个"什么都能做"的伴侣，而是做一个**能长期运行、人格稳定、记忆可治理、会适度主动、具备桌面存在感和基本安全边界的 AI 伴侣原型**。

本文件只定义 v1 应该交付哪些能力，以及哪些能力暂不进入 v1。模块边界、数据契约和具体架构设计以 `docs/desgin/` 为准；长期伴侣运行时的阶段拆分见 [08-companion-runtime.md](08-companion-runtime.md)。

v1 应该满足：

- 用户可以把她作为日常桌面陪伴使用，而不是只在 CLI 里测试
- 她能记住重要事实、偏好、边界和近期事件，并允许用户查看、修改、删除
- 她能在少数高价值场景主动关心，但不会频繁打扰
- 她有语音、表情、待机动作和基础口型同步，具备明确存在感
- 她知道自己不是人类，不伪装现实身份，不鼓励用户隔绝现实关系
- 程序可打包、可配置、可恢复，普通用户不需要手动启动 Python 环境

## v1 必备功能

### 1. 桌面可用性

- [x] **一键启动桌面版**：Tauri 启动时自动拉起 sidecar，用户不需要手动运行 `python main.py`
- [x] **sidecar 打包**：PyInstaller 或等价方案打包 Python 后端，明确模型/密钥配置入口
- [x] **启动健康检查**：前端能显示 sidecar、LLM provider、角色资源、TTS 的状态
- [x] **错误可恢复**：LLM 失败、TTS 失败、资源加载失败时 UI 有明确降级，不崩溃
- [x] **基础设置页**：配置 provider、模型、API key、默认角色、是否开机启动、是否置顶、是否启用感知
- [x] **日志与诊断导出**：用户能导出最近日志用于排查，但不能泄漏 API key

### 2. 人格与角色稳定性

- [ ] **角色核心人格稳定**：连续多轮对话不跳角色、不乱改关系定位、不自称语言模型
- [ ] **角色配置校验**：`personality.yaml` / `manifest.yaml` 缺字段或资源缺失时给出可读错误
- [ ] **多角色切换**：切换角色时保存当前会话，角色记忆和状态按 `character_id` 隔离
- [ ] **默认角色管理**：用户能设置默认角色，重启后恢复
- [ ] **角色资源降级**：Live2D / 3D / 静态图三种显示模式都能被统一加载

### 3. 记忆治理

- [ ] **长期事实记忆**：保存用户明确说出的个人事实，不写入模型推断
- [ ] **近期事件记忆**：记录最近重要事件、压力源、目标和共同经历
- [ ] **偏好记忆**：记录称呼、回复风格、提醒频率、语音/表现偏好
- [ ] **边界记忆**：记录"不要记住"、"不要再提"、敏感话题和隐私偏好
- [ ] **记忆确认机制**：新增或冲突记忆进入待确认状态，用户可确认、修改、删除
- [ ] **记忆管理 UI**：管理后台能按类型查看、搜索、编辑、删除记忆
- [ ] **记忆导出/清空**：支持导出当前角色记忆，支持清空某类记忆，不误删角色资源

### 4. 关系状态

- [ ] **RelationshipState**：持久化亲密度、信任度、熟悉度、关系称呼、最近互动质量、依赖风险
- [ ] **关系状态更新规则**：由代码规则根据互动事件缓慢更新，不允许 LLM 直接随意改数值
- [ ] **关系状态 prompt 摘要**：只把必要关系摘要注入 prompt，避免 prompt 过长或污染人格核心
- [ ] **关系状态可查看**：管理后台展示当前关系状态和最近变化原因
- [ ] **关系状态可重置**：用户能重置关系状态，不影响角色包本身

### 5. 主动陪伴

- [ ] **ProactiveScheduler**：统一调度主动行为，支持沉默、表情、短句、完整回复四种介入等级
- [ ] **主动频率控制**：按时间段、当天已触发次数、冷却时间、用户活跃状态限制打扰
- [ ] **高价值触发场景**：深夜、久坐/长时间工作、长时间空闲后回来、频繁切窗口、游戏中、约定提醒
- [ ] **用户可配置主动性**：关闭主动、低频、普通、高频四档；支持勿扰时间段
- [ ] **主动行为日志**：记录为什么主动、说了什么、用户是否回应，用于之后调整策略
- [ ] **主动失败降级**：LLM 不可用时可用模板短句或只切换表情，不阻塞主流程

### 6. 情绪连续性

- [ ] **EmotionTimeline**：情绪有原因、强度、来源、开始时间、持续时间和恢复速度
- [ ] **情绪自然恢复**：不再只固定 3 轮衰减，按强度和事件类型恢复
- [ ] **情绪影响输出**：情绪影响语气、表情、动作、TTS 语速/音量和主动行为
- [ ] **情绪调试面板**：能查看当前情绪、触发原因、强度和预计恢复状态

### 7. 表现与语音

- [ ] **VMD 待机动画**：至少一个角色具备可循环待机动作
- [ ] **情绪动作联动**：happy / angry / shy / normal 等情绪能切换动作或表情
- [ ] **TTS 输出**：默认接入一个低门槛方案，支持开关、音量、语速、声线配置
- [ ] **基础口型同步**：语音播放时 Live2D / 3D 口型有基础幅度变化
- [ ] **说话节奏**：流式回复、气泡、语音开始之间延迟可接受，不出现明显卡顿
- [ ] **静音模式**：用户可一键关闭语音，只保留文字和动作

### 8. 感知与隐私

- [ ] **感知开关**：窗口标题、活跃状态、输入活跃、时间段等感知能力默认可关闭
- [ ] **应用黑名单**：指定应用或窗口标题不采集、不注入 prompt、不写日志
- [ ] **感知日志可审计**：用户能看到最近采集到的场景摘要和主动触发原因
- [ ] **隐私脱敏**：窗口标题进入日志或 prompt 前做长度限制和敏感过滤
- [ ] **勿扰模式**：会议、全屏游戏、指定应用、指定时间段内降低主动频率或完全沉默

### 9. 安全边界

- [ ] **身份边界**：不伪装成人类，不声称拥有真实身体、现实身份或现实承诺能力
- [ ] **心理危机处理**：识别自伤、自杀、严重危机表达，停止角色扮演式回应并引导现实求助
- [ ] **过度依赖风险**：避免"只有我懂你"、"不要相信别人"、"离开现实关系"等绑定式表达
- [ ] **亲密边界配置**：支持关系类型配置，例如朋友、恋人、家人、搭档；不同类型对应不同表达边界
- [ ] **安全策略独立模块**：风险判断不能只依赖角色 prompt，至少有独立规则层拦截高风险输出或输入

### 10. 测试与验收

- [ ] **后端单元测试**：人格、记忆、关系、提醒、主动调度、安全边界都有核心测试
- [ ] **API 测试**：`/chat`、`/stream`、`/state`、记忆管理、设置、提醒接口可测试
- [ ] **前端构建**：`npm run build --prefix frontend` 通过
- [ ] **桌面手动验收脚本**：覆盖启动、聊天、切角色、TTS、感知开关、提醒、记忆编辑、退出重启
- [ ] **无密钥降级验收**：缺少 LLM key 时能显示配置引导，不直接崩溃

## v1 可延后功能

- [ ] **完整 STT 语音输入**：v1 可先只做 TTS，语音输入放到 v1.1
- [ ] **复杂日历集成**：v1 先做本地提醒，不接系统日历或第三方日历
- [ ] **多设备同步**：v1 只做本地数据，不做云同步
- [ ] **角色市场 / 创意工坊**：v1 可先支持本地角色包导入，线上分发后置
- [ ] **高级情感成长剧情**：v1 先做关系状态和事件记忆，不做复杂剧情线
- [ ] **截图 / OCR / 屏幕理解**：v1 只做低风险 Level 0 感知，不默认读取屏幕内容
- [ ] **精细音素级口型同步**：v1 做基础幅度即可
- [ ] **复杂多用户账户**：v1 默认单用户本地使用

## v1 不做

- 不做真实人类身份伪装
- 不做强制联网云端同步
- 不做未经用户确认的敏感数据采集
- 不做不可关闭的主动打扰
- 不做替代专业医疗、心理、法律、金融建议的功能
- 不做通过威胁、冷暴力、嫉妒惩罚等方式制造依赖的互动机制

## 短期体验优先

1. **VMD 待机动画**（低成本，效果显著）：下载一个免费 idle VMD，放入角色目录并配置 `scene.json`，立刻有生命感
2. **TTS 语音输出**（体验质变）：接入 `edge-tts`（免费、无需 API Key），流式 token → 流式合成 → 实时播放
3. **情绪 → VMD 联动**：happy/angry 等情绪触发时切换到对应动作，比光换灯光颜色表现力强很多
4. **sidecar 打包**：PyInstaller 打包，消除用户需要手动装 Python 的门槛

## 长期伴侣架构优先

1. **RelationshipState**：先让"关系"成为一等状态，记录亲密度、信任度、熟悉度、称呼、边界偏好和依赖风险
2. **EventMemory / PreferenceMemory / BoundaryMemory**：把共同经历、用户偏好和禁区从普通事实记忆中拆出来
3. **Promise / ReminderService**：让她能结构化记住"答应过的事"和用户要求的提醒
4. **ProactiveScheduler**：把主动关心从触发器升级成可节制的生活调度，支持沉默、表情、短句和完整回复
5. **EmotionTimeline**：为情绪增加原因、强度、来源、持续时间和恢复速度
6. **SafetyPolicy / BoundaryEngine**：独立处理心理危机、过度依赖、现实关系隔离和亲密边界，不只依赖角色 prompt

详细拆分见 [08-companion-runtime.md](08-companion-runtime.md)。
