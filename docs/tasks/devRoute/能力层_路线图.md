# 能力层路线图

← [返回总览](开发路线图.md)

设计参考：[能力层设计](../../desgin/能力层.md)

---

## 模块职责

- **负责**：提供 LLM 调用、视觉理解、TTS、Embedding 等基础能力接口
- **不负责**：决定何时调用这些能力（由人格层和感知层约束）
- **约束**：能力层只提供基础能力，不做业务判断

---

## 技术选型

| 功能 | 方案 | 成本估算 |
|------|------|---------|
| 主对话 | `claude-haiku-4-5-20251001` | ~$0.25/1M tokens |
| 视觉理解 | Gemini Flash 2.0 | ~$0.075/1M tokens（后置） |
| TTS（免费） | Edge TTS | 免费（Phase 5） |
| TTS（高质量） | GPT-SoVITS（本地） | 一次性部署（Phase 5） |
| Embedding | text-embedding-3-small | ~$0.02/1M tokens（Phase 3 RAG） |

**月成本估算（个人使用）：每天 50 轮对话 × 3000 tokens ≈ $1.1/月**

## Phase 0：LLM 调用收口

目标：确保 `llm.py` 稳定、可测试、可切换模型。

### 任务清单

- [ ] 确认 `ClaudeClient` 使用 `claude-haiku-4-5-20251001`（成本最低，适合高频对话）
- [ ] 切换模型只改 `llm.py` 中的模型常量，不影响其他层
- [ ] `llm.py` 对 API 错误（`AuthenticationError`、`RateLimitError`、网络超时）给出明确的错误信息，不让异常静默失败
- [ ] 写集成测试（可 mock API 响应）：验证请求格式正确，响应解析正确

### 验收

- 错误的 API Key 给出友好提示，不打印堆栈
- 切换模型只需修改一行常量

---

## Phase 1：成本控制机制

目标：在能力层层面建立成本可见性。

### 任务清单

- [ ] 每次 LLM 调用后，记录 `input_tokens`、`output_tokens` 到日志的 `usage` 字段
- [ ] 提供简单脚本 `tools/cost_report.py`，从日志中汇总 token 消耗和估算成本
- [ ] 加入 `max_tokens` 参数控制（默认 300 token 回复，避免意外长输出）

### 验收

- 每轮对话的 token 消耗可从日志中查询
- `--debug` 模式下显示本轮 token 消耗

## Phase 5：TTS 接入

目标：为角色表现层增强提供语音能力，不阻塞核心对话流程。

### TTS 抽象接口

```python
class BaseTTSBackend:
    def synthesize(self, text: str, emotion: str) -> bytes: ...
    def play(self, audio_bytes: bytes) -> None: ...
```

### 阶段一：Edge TTS（免费，快速验证）

- [ ] 实现 `src/capability/tts/edge_tts_backend.py`
- [ ] 接入 `BaseTTSBackend` 接口
- [ ] 确保 TTS 失败时不阻断对话流程（异步播放，失败静默降级）

### 阶段二：GPT-SoVITS（本地，高质量）

前置条件：有目标角色的语音素材，RTX 显卡部署完成。

- [ ] 实现 `src/capability/tts/sovits_backend.py`
- [ ] 通过 `BaseTTSBackend` 切换，不改上层调用代码
- [ ] 评估本地推理延迟，确保不超过 3 秒（否则严重影响对话节奏）

### 验收

- TTS 开启时角色有语音输出，关闭时静默降级
- 切换 TTS 后端只改配置，不改上层代码

---

## Phase 5 之后：视觉理解（后置）

前置条件：Level 0 感知无法判断场景时才触发，成本控制机制已验证。

- [ ] 实现 `src/capability/vision.py`：`analyze_screenshot(image_bytes: bytes) -> str`，使用 Gemini Flash
- [ ] 只在感知层明确请求时调用，不自动触发
- [ ] 每次调用记录成本，避免频繁触发

---

## 成本控制原则

- 默认每轮只走低成本主对话模型（Haiku）
- 只有基础感知无法判断时，才调用更重的视觉能力
- 记忆、RAG、感知结果不能无节制增加 prompt 长度
- 摘要生成（Phase 2）使用同一个 Haiku 模型，不升级模型
