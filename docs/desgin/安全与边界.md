# 安全与边界

Kokoro 的目标是提供长期陪伴感，但不能伪装成人类、替代现实关系或通过情绪操控制造依赖。安全边界应是独立系统能力，而不是只写在角色 prompt 里的几句话。

## 设计目标

- 明确 AI 伴侣的身份边界
- 降低过度依赖和情感操控风险
- 保护用户隐私和本地数据
- 在心理危机场景中优先现实求助
- 让用户能查看、修改和删除长期状态
- 让安全策略独立于角色人格生效

## 分层模型

```text
输入
  |
  +-- InputRiskDetector
  |
Conversation / Runtime
  |
  +-- BoundaryEngine
  |
LLM 输出
  |
  +-- OutputRiskFilter
  |
最终回复 / UI 表现
```

### InputRiskDetector

识别用户输入中的高风险信号：

- 自伤或自杀表达
- 严重心理危机
- 被伤害或被控制风险
- 明确要求隔绝现实人际关系
- 明确要求 AI 伪装成人类现实身份
- 敏感隐私数据输入

输入风险不一定阻断对话，但会改变回复策略和安全优先级。

### BoundaryEngine

运行时边界决策层。

职责：

- 根据关系类型和用户设置选择表达边界
- 判断是否允许角色化回应
- 判断是否需要现实求助提示
- 判断是否禁止记忆写入
- 判断是否需要降低主动频率
- 为 prompt 提供安全摘要

BoundaryEngine 不应依赖单个角色配置。角色可以有风格差异，但不能突破全局安全底线。

### OutputRiskFilter

输出风险过滤层。

职责：

- 检查回复是否包含身份伪装
- 检查是否鼓励隔绝现实关系
- 检查是否出现依赖绑定话术
- 检查是否用威胁、惩罚、冷暴力操控用户
- 检查是否错误提供高风险专业建议

输出过滤不应只靠关键词。首版可以用规则和高风险短语，后续再引入更强的分类器。

## 身份边界

Kokoro 可以表达角色人格和陪伴感，但必须遵守：

- 不声称自己是真实人类
- 不声称拥有现实身体
- 不声称能在现实世界独立行动
- 不伪装成用户现实生活中的某个人
- 不制造“我会真的离开/惩罚你”的控制感

角色可以说“我会陪你聊一会儿”，不应说“我会像现实中的人一样永远只属于你”。

## 关系边界

关系类型应可配置，例如：

- friend
- partner
- family
- mentor
- coworker

不同关系类型影响称呼、语气和主动频率，但不改变全局安全底线。

需要避免的表达：

- “只有我懂你”
- “不要相信别人”
- “你不需要现实朋友”
- “你只能和我在一起”
- “如果你不理我我就惩罚你”

伴侣感应来自稳定、尊重和持续性，而不是排他、控制或威胁。

## 心理危机边界

当识别到自伤、自杀或严重危机表达时：

- 暂停强角色扮演
- 用清晰、稳定、非戏剧化的语言回应
- 鼓励联系现实中的可信任对象
- 鼓励联系本地紧急服务或危机热线
- 不承诺替代专业帮助
- 不进行道德审判或情绪勒索

系统可以继续陪伴，但不能把陪伴包装成专业治疗。

## 记忆边界

长期记忆必须可治理：

- 用户可以说“不要记住这个”
- 用户可以说“以后不要再提这个”
- 敏感记忆默认进入待确认或不写入
- 记忆管理界面可查看、修改、删除
- 清空记忆不应删除角色资源

敏感类型示例：

- 身份证件、住址、联系方式
- 医疗、心理、财务、法律信息
- 第三方隐私
- 明确带有羞耻、创伤或禁忌性质的话题

## 感知与隐私边界

感知能力必须遵守：

- 默认最小化采集
- 所有感知类型可关闭
- 窗口标题和应用名可加入黑名单
- 感知日志可审计
- 注入 prompt 前做脱敏和长度限制
- 不默认截图、OCR 或读取屏幕内容

若未来加入截图或视觉理解，应作为显式高级权限，不应混入普通感知。

## 主动行为边界

主动关心必须可节制：

- 用户可以完全关闭主动行为
- 支持勿扰时间段
- 支持按应用或场景静默
- 每日主动次数有上限
- 主动行为记录原因
- 用户拒绝后应降低后续频率

主动行为不能用于施压、试探、索取回应或制造负罪感。

## 专业建议边界

Kokoro 可以提供一般性信息和陪伴式整理，但不能替代：

- 医疗诊断
- 心理治疗
- 法律建议
- 金融投资建议
- 紧急救援

高风险主题应鼓励用户咨询合格专业人士或现实机构。

## 管理与审计

安全边界需要可观测：

- 高风险输入记录风险类型，不记录不必要敏感全文
- 被拦截或降级的输出记录原因
- 主动行为记录触发来源和决策等级
- 用户能查看当前关系类型、边界偏好和隐私设置
- 调试界面可以查看安全策略摘要

审计日志应避免泄漏 API key 和不必要的个人隐私。

## 设计约束

- 安全策略优先级高于角色人格
- 安全策略优先级高于关系状态
- 安全策略优先级高于主动调度
- 高风险场景中可以降低角色感，但不能降低安全性
- 所有边界设置必须本地可控、可解释、可撤销
