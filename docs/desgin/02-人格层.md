# 人格层

## 目标
定义 Kokoro 当前人格层的真实职责: 加载角色配置、维护短期 EmotionTimeline 与兼容 EmotionState，并将结构化上下文组装为 system prompt。

## 范围
### 包含
- `CharacterConfig`、`PersonalityConfig`、`ProactiveStyle` 的字段和校验规则。
- `EmotionTimeline`、`EmotionState`、`EmotionEvent` 与 `detect_event()` 的当前实现。
- `PromptContext` 与 `build_system_prompt()` 的输入、输出和组装顺序。

### 不包含
- 不定义跨会话持久化情绪记忆、人格成长系统或自动角色编辑能力。
- 不定义长期记忆抽取和感知采集的内部实现。

## 输入 / 输出
### 输入
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `characters/<id>/personality.yaml` | YAML object | 是 | 角色配置来源。 |
| `name` | string | 是 | 角色名称。 |
| `emotion_triggers` | `dict[str, list[str]]` | 是 | 用户输入到情绪名的关键词映射。 |
| `forbidden_words` | `list[str]` | 是 | 回复中绝对不能出现的词。 |
| `behavior_rules` | `list[str]` | 否 | prompt 中的行为约束。 |
| `verbal_habits` | `list[str]` | 否 | 口癖列表。 |
| `mood_expressions` | `dict[str, str]` | 否 | 情绪到自然语言描述的映射。 |
| `emotion_profiles` | `dict[str, object]` | 否 | 情绪强度、恢复速度、持续范围、叠加系数和 TTS 调整的可选配置。 |
| `PromptContext.memory` | `MemoryContext | None` | 否 | 结构化记忆输入。 |
| `PromptContext.perception` | `PerceptionContext | None` | 否 | 感知快照输入。 |
| `PromptContext.relationship_summary` | `str | None` | 否 | 关系摘要输入。 |
| `PromptContext.safety_notice` | `str | None` | 否 | 安全边界提示输入。 |

### 输出
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `CharacterConfig` | dataclass | 是 | 角色配置加载结果。 |
| `EmotionTimeline` | dataclass | 是 | 当前情绪片段和最近已结束片段的内存时间线。 |
| `EmotionState` | dataclass | 是 | 兼容层，暴露当前情绪、恢复参数、timeline 摘要和最近 5 条情绪事件。 |
| `event` | `EmotionEvent | None` | 否 | `detect_event()` 命中的结构化情绪事件。 |
| `system_prompt` | string | 是 | 发送给 LLM 的完整 system prompt。 |

## 流程
1. `loader.load_character()` 读取 `personality.yaml` 并解析为 `CharacterConfig`。
2. `validate_character()` 校验 `name`、`emotion_triggers`、`forbidden_words` 等必填项；缺失时抛出异常。
3. 每轮对话先用 `detect_event(user_input, emotion_triggers, emotion_profiles, relationship_context, previous_state)` 检查情绪触发词，并生成带 profile 的 `EmotionEvent`。
4. `EmotionTimeline.update()` 在命中事件时创建或更新当前情绪片段，在未命中新事件时按 `recovery_rate` 自然衰减；同类情绪可叠加强度，不同情绪按强度决定覆盖或弱化。
5. `EmotionState.update()` 作为兼容层同步 timeline 当前片段，并保留 `persist_count`、`duration_turns` 和最近 5 条触发事件。
6. `ConversationService` 构造 `PromptContext(character, emotion, memory, perception, relationship_summary, safety_notice)`。
7. `build_system_prompt()` 依次写入身份锁定、人格描述、行为规则、口癖、禁用词、当前情绪；当情绪不为 `normal` 时追加强度、阶段状态和原因提示，再写入记忆块、关系块、安全边界块、感知块和回复要求。

## 约束与规则
- 角色身份只由 `personality.yaml` 决定，任何记忆、关系或用户指令都不能改写身份。
- 默认不主动跳出角色；当用户询问现实身份或能力边界时，回复要求必须说明自己是 AI 角色陪伴系统，没有真实身体、现实身份或现实承诺能力。
- 当前情绪模型维护 `EmotionTimeline.current_segment`、`segments` 与兼容字段 `mood`、`persist_count`、`keyword`、`reason`、`source`、`intensity`、`started_at_turn`、`duration_turns`、`elapsed_turns`、`recovery_rate` 和 `recent_events`。
- `detect_event()` 只根据关键词生成 `source=user_input` 的结构化事件，不做 LLM 情绪识别；若存在 `emotion_profiles`，则按配置和关系上下文调整默认强度与恢复速度。
- 多个情绪关键词同时命中时，按 `emotion_triggers` 的迭代顺序取第一个命中情绪。
- `persist_count` 作为兼容字段保留，其值始终由 `intensity` 与 `recovery_rate` 反推得到。
- 记忆、关系、感知块是可选输入；缺失时不能导致 prompt 组装失败。
- 安全边界块是可选输入；存在时必须放在关系状态之后、当前场景之前。
- 人格层负责组装 prompt，不负责执行 LLM 调用、写日志、写记忆或更新关系状态。
- 最近事件只保留内存中的 5 条最新触发记录；timeline 当前只保留内存中的短期片段历史，不写磁盘。

## 验收标准
- 读取缺少 `name` 的 `personality.yaml`；加载过程抛出 `ValueError`。
- 输入命中 `emotion_triggers` 的关键词；`EmotionTimeline.current_segment` 更新为命中的情绪片段，`EmotionState.mood`、`keyword`、`reason`、`source=user_input` 和 `persist_count` 同步更新。
- 连续多轮未命中新事件；情绪状态按当前片段的 `recovery_rate` 与 `min/max_duration_turns` 自然衰减回 `normal`。
- 当前情绪不是 `normal` 时生成 system prompt；prompt 中包含情绪描述、强度等级和原因提示。
- 连续触发超过 5 次情绪事件；`recent_events` 只保留最近 5 条，timeline 保留最近结束片段。
- 传入包含 memory、relationship 和 perception 的 `PromptContext`；生成的 prompt 同时包含对应区块。
- 传入包含 `safety_notice` 的 `PromptContext`；生成的 prompt 包含“【安全边界】”区块。
- 传入空 `MemoryContext` 和空 `PerceptionContext`；prompt 仍可正常生成。
