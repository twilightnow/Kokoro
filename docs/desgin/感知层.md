# 感知层

感知层负责采集有限、本地、可关闭的环境信号，并输出结构化场景。它让 Kokoro 知道“现在大概是什么情境”，但不直接生成回复，也不直接决定是否打扰用户。

## 职责

感知层负责：

- 采集系统时间
- 采集活动窗口标题或应用名
- 采集用户活跃/空闲状态
- 统计窗口切换频率
- 粗略识别游戏、工作、休息、深夜等场景
- 输出 `PerceptionContext`
- 写入可审计的感知摘要

感知层不负责：

- 生成主动台词
- 判断是否应该主动介入
- 读取屏幕内容
- 保存长期关系状态
- 修改情绪或记忆

## 输入源

默认 Level 0 感知只包含：

- 当前时间
- 活动窗口标题
- 活动应用名
- 用户是否活跃
- 空闲时长
- 窗口切换频率

默认不包含：

- 截图
- OCR
- 摄像头
- 麦克风持续监听
- 文件内容读取
- 浏览器页面内容读取

更高等级感知必须是显式权限，不应默认开启。

## PerceptionContext

建议上下文结构：

- `time_of_day`
- `hour`
- `is_late_night`
- `active_window_title`
- `active_app_name`
- `is_user_active`
- `idle_seconds`
- `switches_per_minute`
- `is_gaming`
- `activity_type`
- `privacy_filtered`

其中 `activity_type` 是高层分类结果，例如：

- `working`
- `gaming`
- `watching_video`
- `idle`
- `late_night_active`
- `unknown`

## PrivacyFilter

感知信息进入日志或 prompt 前必须经过隐私过滤。

过滤规则：

- 应用黑名单命中时不采集或不注入
- 窗口标题长度限制
- 敏感关键词脱敏
- 隐私模式下只保留高层分类
- 不记录完整敏感标题

示例：

```text
原始窗口标题：某银行账户明细 - 浏览器
注入场景：用户正在使用被标记为敏感的应用，具体标题已隐藏
```

## ActivityClassifier

活动分类器把低层信号转为高层场景。

输入：

- 时间段
- 应用名
- 窗口标题
- 活跃状态
- 切换频率
- 用户配置

输出：

- 活动类型
- 置信度
- 是否适合主动
- 是否需要隐私脱敏

分类器只给出场景，不决定最终行为。是否主动由 `ProactiveScheduler` 决定。

## 触发器与调度器边界

触发器负责发现“可能值得关注的事件”。

示例：

- 深夜仍然活跃
- 长时间空闲后回来
- 连续工作太久
- 频繁切窗口
- 游戏中

触发器输出 `PerceptionEvent`。

`ProactiveScheduler` 再结合关系状态、用户偏好、勿扰设置、当天触发次数和提醒事件，决定是否介入以及介入等级。

## 感知日志

感知日志用于审计，不用于监控用户。

建议记录：

- 触发时间
- 触发类型
- 脱敏后的场景摘要
- 是否进入主动调度
- 最终介入等级
- 用户是否回应

不应记录：

- 完整敏感窗口标题
- API key
- 未经允许的屏幕内容
- 与触发无关的大量行为轨迹

## 用户控制

用户必须能配置：

- 是否启用感知
- 启用哪些感知类型
- 应用黑名单
- 勿扰时间段
- 全屏/会议/游戏时是否沉默
- 是否写入感知日志
- 是否将感知上下文注入 prompt

## 设计约束

- 感知默认最小化
- 感知失败必须静默降级
- 感知层只输出结构化上下文
- 敏感信息先过滤再注入
- 高级感知需要显式授权
- 任何感知能力都必须可关闭
