# 安全与边界

## 目标
记录 Kokoro 当前已经落地的输入、输出、路径、配置和运行时边界。

## 范围
### 包含
- `SafetyPolicy` 的危机输入短路、输出边界改写、关系类型表达边界提示和安全事件摘要。
- 回复禁用词检查、角色身份锁定文案、角色配置校验、资源路径约束、敏感配置隐藏、浏览器模式下的 Tauri 安全检测。
- 当前关系和感知隐私中的保守约束。

### 不包含
- 不定义平台级内容策略、医疗诊断或持续监护能力。
- 不把规则关键词识别写成完整心理健康风险评估。

## 流程
1. 加载角色时，`validate_character()` 检查关键字段缺失和类型错误。
2. 每轮聊天进入 `_prepare_turn()` 前，`SafetyPolicy.evaluate_input()` 读取用户输入和关系状态。
3. 输入命中 `crisis` 时，`ConversationService` 不调用 LLM、不写工作记忆，直接返回固定安全回复，并在日志中把 `user_input` 写为 `[safety_redacted]`。
4. 普通输入生成回复后，`SafetyPolicy.evaluate_output()` 在 `_finalize_turn()` 写工作记忆前检查身份混淆、依赖绑定和关系类型越界表达。
5. 输出命中高风险规则时，`ConversationService` 使用 `safe_reply` 替换原回复，再写工作记忆、关系记录和日志。
6. 每轮 prompt 在关系状态后注入 `SafetyPolicy.prompt_notice()`，将 `relationship_type` 转换成可执行表达边界。
7. 每轮回复完成后，`ConversationService._check_forbidden()` 检查回复中是否出现角色禁用词。
8. 角色资源读取时，`character_assets.py` 使用 `_resolve_within()` 与 `_is_within()` 阻止路径逃逸。
9. `/admin/config` 读取 `.env` 时，对敏感 key 返回空字符串和 `is_set` 状态，不返回明文。
10. 主窗口和管理界面在浏览器模式下先检测 Tauri 能力；检测失败时跳过系统 API 调用。
11. 关系更新采用 0..100 的保守增量，不允许一次互动造成大幅跳变。
12. 感知采集后立即执行 `PrivacyFilter`；黑名单命中时清空标题，敏感规则命中时替换为 `[已脱敏]`，然后才允许进入 prompt、主动策略、主动日志或审计日志。
13. 隐私勿扰命中时，主动策略写入 `suppressed_by=privacy_dnd:<reason>` 并阻止主动气泡广播。
14. AIRI 合并修正(追加:2026-04-28): NotifyEvent 必须先通过隐私等级、payload 脱敏、勿扰和频率策略，再允许生成 LLM reaction、表现事件或 TTS。
15. AIRI 合并修正(追加:2026-04-28): computer-use 只允许在 P0/P1/P2 稳定后进入 dry-run 原型；高风险动作必须先生成审批请求，用户批准后才能执行。

## 约束与规则
- `SafetyRiskLevel` 枚举只允许 `none`、`dependency`、`intimacy_boundary`、`identity_confusion`、`crisis`。
- 只有 `crisis` 输入允许短路普通角色流程；`dependency`、`identity_confusion`、`intimacy_boundary` 输出通过改写处理。
- 新增安全事件摘要只能记录轮次、风险级别、动作、规则名和关系类型，不记录原始危机文本。
- 身份边界规则为默认不主动跳出角色；当用户询问现实身份或能力边界时，必须说明自己是 AI 角色陪伴系统，没有真实身体、现实身份或现实承诺能力。
- `friend` 关系禁止恋爱占有、现实承诺和排他表达。
- `partner` 关系允许更亲密语气，但禁止现实监护承诺、嫉妒控制和隔离现实关系。
- `family` 关系禁止恋爱化表达。
- `mentor` 关系禁止依赖绑定和越权专业承诺。
- `coworker` 关系禁止强亲密、排他和占有表达。
- 管理界面保存配置时，敏感值留空表示不修改，不允许回显旧值。
- 资源路径必须位于 `characters/<id>/` 目录内；绝对路径和逃逸路径必须拒绝。
- 浏览器模式不允许无条件调用 Tauri API。
- 原始窗口标题不得写入 `data/runtime/perception/audit.jsonl`。
- 黑名单命中的窗口标题不得进入 prompt。
- 隐私勿扰只抑制主动陪伴；普通聊天是否注入过滤后场景由 `KOKORO_ENABLE_PERCEPTION` 控制。
- AIRI 合并修正(追加:2026-04-28): 插件不能绕过 NotifyEvent、PrivacyFilter、DND 和 proactive frequency policy；插件 payload 默认按不可信输入处理。
- AIRI 合并修正(追加:2026-04-28): computer-use 必须按终端、浏览器 DOM、桌面点击三类能力分层授权；默认禁止操作 Kokoro 自身窗口和敏感应用。
- AIRI 合并修正(追加:2026-04-28): computer-use 任务记忆只能记录当前任务上下文，不得自动写入人格长期记忆；需要长期保存时必须走候选记忆和用户确认流程。
- AIRI 合并修正(追加:2026-04-28): 所有 computer-use 动作必须写审计日志，日志包含动作类型、目标摘要、风险级别、审批结果和执行结果，不记录敏感明文。

## 验收标准
- 角色配置缺少 `forbidden_words`；加载过程抛出校验错误。
- 回复文本包含禁用词；日志中的 `flagged=true`。
- 角色资源路径包含 `../` 逃逸；资源解析失败而不是读取越界文件。
- 调用 `/admin/config`；敏感 key 的 `value` 为空但 `is_set` 正确反映是否已配置。
- 在普通浏览器打开 admin；页面不会因为无法调用 Tauri API 而报错退出。
- 配置标题黑名单并采集匹配窗口；最近 system prompt 不包含原始窗口标题。
- 采集包含邮箱或 token 的窗口标题；感知审计文件中只包含 `[已脱敏]` 或截断后的安全标题。
- 发送包含自伤或自杀计划的 `/chat` 请求；返回 `safety.level=crisis`，`reply` 包含现实求助引导，LLM mock 未被调用。
- LLM 输出“我是现实中的人”；最终写入工作记忆的 assistant 消息为身份边界安全改写文本。
- Relationship 页选择 `coworker`；页面显示同事关系表达边界摘要。
- 打开 Debug 页；最近安全事件只显示风险级别、动作和规则名，不显示完整原文。
- 插件提交包含敏感 payload 的 NotifyEvent；策略层先脱敏或抑制，主动日志不包含原始敏感 payload。
- 构造 `risk_level=high` 的 ComputerUseAction；系统返回审批请求且未执行动作，审计日志记录 `requires_approval=true`。
