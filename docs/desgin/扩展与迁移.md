# 扩展与迁移

所属项目：[[Kokoro]]

---

## 核心设计：角色包（Character Package）

角色不是一个配置文件，而是一个完整的可安装包。

但角色包属于扩展能力，不是首版前置条件。首版只需要支持从本地目录加载单角色。

```
characters/
└── asuka/                   # 包目录名 = 角色ID
    ├── manifest.yaml        # 包元数据（名称、版本、依赖、作者）
    ├── personality.yaml     # 人格配置
    ├── live2d/              # Live2D 模型文件
    │   ├── asuka.model3.json
    │   └── ...
    ├── voice/               # 语音素材（用于TTS克隆）
    │   └── samples/
    ├── corpus/              # 台词库
    │   └── lines.jsonl
    └── preview.png          # 预览图
```

**manifest.yaml 结构：**

```yaml
id: "asuka-langley-tv"
name: "惣流·明日香·兰格雷"
version: "1.0.0"
author: ""
description: ""
license: "CC BY-NC-SA"       # 明确声明素材授权
requires:
  platform: ">=0.1.0"
  live2d_model: true
  tts_voice: true
```

## 创意工坊机制

**本地工坊（不依赖中心服务器）：**

```
用户分享角色包
    │
    ▼
导出为 .charpkg 文件（实际是 zip）
    │
    ▼
其他用户拖入应用窗口 / 命令行安装
    ▼
platform install ./asuka.charpkg
```

首版不需要完整工坊流程，保留目录加载和简单导入即可。

**插件接口（后续扩展）：**

触发器、记忆处理器、TTS后端均设计为可插拔接口，第三方可实现自定义插件：

```python
# 触发器插件示例
class CustomTrigger(BaseTrigger):
    def check(self, context: PerceptionContext) -> bool: ...
    def generate_prompt(self, state: EmotionState) -> str: ...
```

插件以 Python 包形式安装，通过 `platform.plugins` 目录自动发现。

## 迁移性设计

**用户数据完全集中在一个目录：**

```
~/ai-agent-platform/         # 用户数据根目录（可自定义路径）
├── characters/              # 已安装的角色包
├── memories/                # 各角色的记忆数据库
│   └── asuka/
│       ├── chroma/          # 向量数据库
│       └── relations.db     # SQLite 关系状态
├── settings.yaml            # 全局设置
└── logs/
```

**迁移操作：**

```bash
# 备份：打包整个用户目录
platform export --output backup.zip

# 恢复：解压到新机器
platform import backup.zip

# 仅迁移某个角色（含记忆）
platform export --character asuka --output asuka_full.zip
```

**约束：**
- 应用内无任何硬编码路径
- 所有路径相对于用户数据根目录
- 数据库文件使用相对路径存储引用
- 不写注册表（Windows）/ 不写 `/etc`（Linux/Mac），保持绿色

## 对各层的影响

| 层 | 影响 |
|----|------|
| 人格层 | YAML 拆分为 manifest + personality，角色ID成为全局唯一标识 |
| 记忆层 | ChromaDB collection 命名以角色ID为前缀，路径可配置 |
| UI层 | 支持拖拽安装 .charpkg，工坊浏览界面（后期） |
| 感知层 | 无影响 |
| 能力层 | TTS后端抽象为接口，允许角色包携带自定义TTS配置 |

补充约束：

- 首版先保证目录结构稳定，再考虑 `.charpkg` 封装
- 迁移能力优先覆盖角色配置、对话摘要、长期记忆
- 插件接口和创意工坊均后置，避免过早扩展
