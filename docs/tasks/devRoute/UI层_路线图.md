# UI 层路线图

← [返回总览](开发路线图.md)

设计参考：[UI层设计](../../desgin/UI层.md)

---

## 模块职责

- **负责**：桌面壳、角色表现层（立绘/Live2D）、系统托盘、输入框、气泡展示
- **不负责**：人格判断、记忆存储、感知采集
- **约束**：UI 只负责展示和交互，所有角色逻辑由 Python 侧决定

---

## 技术选型

| 组件 | 方案 | 许可证 |
|------|------|--------|
| 客户端壳 | Tauri | MIT |
| 前端框架 | Vue 3 | MIT |
| Live2D 渲染 | pixi-live2d-display | MIT |
| 3D 立绘备选 | three-vrm | MIT |

**运行时架构**：Tauri → IPC → Python FastAPI sidecar（Python 侧负责核心逻辑，前端只做展示）

---

## 表现层演进路径（从轻到重）

| 形式 | 适合阶段 | 说明 |
|------|---------|------|
| 纯文字气泡 | Phase 0–1 | 无立绘，仅 CLI |
| 静态立绘 + 表情差分 | Phase 4 过渡 | PNG 切换，情绪对应不同表情图 |
| 漫画风格帧动画 | Phase 4–5 | 少量帧表现关键动作，轻量 |
| Live2D 骨骼动画 | Phase 5 | 流畅自然，当前主选方案 |
| 3D 模型（VRM） | Phase 5 后 | 自由度最高，备选 |

---

## Phase 0–3：UI 暂不启动

CLI 阶段不引入 Tauri 和 FastAPI sidecar。此阶段 UI 层任务：

- [ ] 确认 Python 核心逻辑（人格层、记忆层、感知层）不依赖任何 UI 框架
- [ ] 设计 IPC 协议草案（定义消息格式，不实现），便于 Phase 4 接入

---

## Phase 4：桌面运行时与 UI 骨架

目标：把 CLI 原型升级为真正的桌面常驻应用，保持"低打扰"和"人格优先"。

### 4.1 Python 侧 FastAPI Sidecar

任务：

- [ ] 在 Python 侧实现 `src/server/app.py`：FastAPI 应用，暴露以下端点：
  - `POST /chat`：接收用户输入，返回角色回复 + 情绪状态
  - `GET /state`：返回当前情绪状态和活跃场景
  - `POST /trigger`：外部触发主动介入（感知层调用）
- [ ] Sidecar 以子进程方式由 Tauri 启动，Tauri 关闭时自动停止
- [ ] 确保 CLI 入口（`main.py`）与 FastAPI 共用同一套核心逻辑，不重复实现

### 4.2 IPC 协议定义

消息格式（JSON）：

```json
// 前端 → 后端
{ "type": "user_input", "text": "用户输入内容" }

// 后端 → 前端
{
  "type": "reply",
  "text": "角色回复",
  "emotion": "happy",
  "quick_responses": ["嗯", "知道了", "说下去"]
}

// 后端 → 前端（主动介入）
{
  "type": "proactive",
  "text": "一句主动介入的话",
  "trigger": "late_night",
  "quick_responses": ["好的", "没事"]
}
```

任务：

- [ ] 在 `docs/` 中写一份 IPC 协议文档
- [ ] 前端和后端严格遵循此协议，不在协议之外传递人格逻辑

### 4.3 Tauri 桌面壳

任务：

- [ ] 初始化 Tauri + Vue 3 项目（位于 `frontend/` 目录）
- [ ] 配置桌面悬浮窗：无边框、透明背景、置顶可选
- [ ] 配置系统托盘：右键菜单包含"打开聊天"、"切换锁定模式"、"退出"
- [ ] 实现点击穿透：立绘区域可点击，透明区域穿透到底层窗口

### 4.4 基础 UI 组件

#### 气泡展示（漫画对白框范式）

- 每次只显示当前这句话，说完后自动淡出
- 无历史堆叠，无滚动条
- 气泡出现 / 消失有轻微动画，不是瞬切

任务：

- [ ] 实现 `BubbleComponent`：接收文本，显示后 N 秒自动淡出
- [ ] 实现流式输出效果：逐字或逐词显示，比一次性弹出更像在说话
- [ ] 实现"思考指示"：LLM 调用期间显示省略号动画

#### 输入框

- 默认不显示
- 用户点击立绘或双击进入对话模式后，输入框淡入
- 她回复完毕后，输入框保持 N 秒等待，无操作则淡出

任务：

- [ ] 实现 `InputBoxComponent`：支持淡入/淡出，发送后清空
- [ ] 实现快捷回应按钮：预设短回复，一键发送

#### 窗口模式切换

| 模式 | 说明 |
|------|------|
| 锁定模式 | 常驻陪伴，纯角色展示，隐藏输入框，仅响应主动介入和轻量交互 |
| 对话模式 | 用户主动点击后进入，展示输入框，支持完整对话 |

任务：

- [ ] 实现双击切换对话模式 / 锁定模式
- [ ] 短按（< 300ms）触发轻交互，长按拖动移动位置
- [ ] 锁定模式下保留点击和快捷回应

#### 窗口行为

任务：

- [ ] 默认位置：屏幕右侧边缘居中（避开任务栏和通知区）
- [ ] 拖动到屏幕边缘时自动吸附，露出立绘一半
- [ ] 吸附状态下点击或气泡出现时自动滑出
- [ ] 点击 / 拖动 / 双击行为区分（300ms 阈值）

### 4.5 静态立绘接入（Phase 4 过渡表现）

任务：

- [ ] 按情绪准备 PNG 差分图（`normal` / `happy` / `angry` / `shy` / `cold`）
- [ ] 实现 `CharacterDisplayComponent`：根据后端返回的 `emotion` 字段切换立绘图片
- [ ] 确保立绘切换不影响对话流程

### 验收

- 应用可常驻，不影响正常桌面操作
- 锁定模式下立绘常驻，气泡在主动介入时出现后自动消失
- 对话模式下输入框可用，回复以流式气泡呈现
- 核心逻辑可不启动 UI 单独测试（CLI 入口仍可用）

## Phase 5：角色表现层增强

### 5.1 漫画风格帧动画（可选过渡）

任务：

- [ ] 增加眨眼帧动画（循环）
- [ ] 说话时口型动画（简单帧切换）
- [ ] 情绪切换时加过渡动画（不是瞬切）

### 5.2 Live2D 接入

前置条件：有目标角色的 Live2D 模型文件（需确认使用条款）。

任务：

- [ ] 引入 `pixi-live2d-display`
- [ ] 实现情绪对应 Live2D 表情参数映射
- [ ] 确保 Live2D 渲染不影响气泡和输入框的交互
- [ ] 保留静态立绘作为降级方案（Live2D 模型缺失时降级）

### 5.3 主动介入出现方式

目标：主动介入不应直接弹出气泡，应有前置提示。

任务：

- [ ] 实现"引起注意"动画：立绘轻微动作（抬头、眨眼加速）
- [ ] 动作后停顿约 1 秒，再弹出气泡
- [ ] 气泡附带快捷回应按钮
- [ ] 用户无响应 N 秒后气泡淡出，不强迫回应

### 验收

- Live2D / 静态立绘 / 纯文字气泡可通过配置切换，不改代码
- 主动介入时"引起注意"动画先于气泡出现
- TTS 开启时角色有语音输出，关闭时静默降级

## 依赖说明

| 依赖 | 来源 | 说明 |
|------|------|------|
| FastAPI sidecar | Python 核心 | Phase 4 前 Python 侧必须暴露 HTTP 接口 |
| 情绪状态 | 人格层 | 通过 IPC `/state` 端点获取 |
| 主动介入事件 | 感知层 → 人格层 | 通过 IPC `proactive` 消息推送到前端 |
| Live2D 模型 | 角色包 | Phase 5，需确认使用条款 |

---

## 不建议在此阶段做的事

- Phase 4 之前启动 Tauri 或 Vue（CLI 阶段人格层未稳定，UI 先行无意义）
- Live2D 优先于静态立绘（静态立绘是更低成本的验证方案）
- 历史记录默认展示（气泡范式不堆叠历史，查阅历史通过托盘菜单单独打开）
- 语音输入优先于 TTS（Phase 5 之后）
