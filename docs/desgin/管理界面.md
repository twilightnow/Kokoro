# 管理界面

管理界面是 Kokoro 的本地控制台，用于配置、查看、治理和排障。它不是普通聊天界面，也不应承载角色陪伴体验。

## 设计目标

- 提供当前运行状态的可观测性
- 管理角色、记忆、关系、提醒、边界和配置
- 提供调试入口，但不绕过核心服务边界
- 让用户能控制长期状态和隐私设置
- 让高风险能力有可见开关和审计入口

## 信息架构

建议管理界面分成以下区域：

- 状态总览
- 角色管理
- 记忆管理
- 关系状态
- 提醒与承诺
- 情绪与主动行为
- 感知与隐私
- 安全边界
- 日志与诊断
- 设置
- 调试工具

## 状态总览

展示当前系统是否可用。

核心信息：

- sidecar 健康状态
- 当前角色
- 当前情绪
- 当前关系摘要
- LLM provider 状态
- TTS 状态
- 感知开关状态
- 主动行为模式
- 最近错误
- 当前会话 token 用量

总览只做展示和跳转，不承载复杂编辑。

## 角色管理

职责：

- 列出可用角色
- 查看角色配置解析结果
- 切换当前角色
- 设置默认角色
- 查看 display 能力
- 查看资源加载状态

设计边界：

- 编辑角色 YAML 是开发能力，应有明确提示
- 角色包安装和用户数据删除必须区分
- 角色切换应先保存旧会话状态

## 记忆管理

职责：

- 按类型查看记忆
- 搜索记忆
- 编辑或删除记忆
- 处理待确认记忆
- 导出或清空记忆

记忆类型：

- facts
- summaries
- preferences
- boundaries
- events
- promises

设计要求：

- 敏感记忆有标记
- 冲突记忆有来源和待确认状态
- 删除操作需要明确确认
- 清空记忆不删除角色资源

## 关系状态

职责：

- 查看 `RelationshipState`
- 查看最近变化原因
- 修改关系类型
- 调整称呼偏好
- 重置关系状态

设计边界：

- 不提供任意大幅修改隐藏数值的娱乐化入口
- 数值变化应可解释
- 重置关系不影响长期事实记忆，除非用户明确选择

## 提醒与承诺

职责：

- 查看当前提醒
- 新增提醒
- 完成、跳过、取消提醒
- 查看角色答应过的待跟进事项
- 查看周期规则

设计边界：

- 提醒是结构化数据，不是普通摘要
- 过期提醒需要明确状态
- 提醒触发应进入主动调度，而不是直接弹窗

## 情绪与主动行为

职责：

- 查看当前情绪和强度
- 查看情绪原因
- 查看主动行为日志
- 配置主动频率
- 配置勿扰时间段
- 临时暂停主动行为

设计边界：

- 调试情绪注入只能作为开发工具
- 用户配置优先于角色主动风格
- 主动行为必须可关闭

## 感知与隐私

职责：

- 开关感知能力
- 配置应用黑名单
- 配置窗口标题脱敏
- 查看感知日志摘要
- 配置是否注入 prompt

设计边界：

- 不展示未经脱敏的大量原始窗口标题
- 高级感知能力必须单独授权
- 隐私设置应在主窗口和管理界面都容易找到

## 安全边界

职责：

- 配置关系类型
- 查看当前边界偏好
- 查看被拦截或降级的安全事件
- 配置敏感话题
- 查看依赖风险摘要

设计边界：

- 安全底线不可被角色配置关闭
- 用户可以调整互动边界，但不能关闭危机处理
- 安全日志应避免泄漏敏感全文

## 日志与诊断

职责：

- 查看会话日志
- 查看错误日志
- 导出脱敏诊断包
- 查看最近 system prompt 摘要
- 查看 provider 错误

设计要求：

- API key 永不回显
- 导出默认脱敏
- 清空日志需要显式确认
- 日志查看不应成为隐私泄漏源

## 设置

职责：

- LLM provider
- model
- API key 设置状态
- TTS 声音和开关
- 默认角色
- 数据目录
- 开机启动
- 窗口行为
- 感知和主动默认策略

设计边界：

- 敏感 key 只显示是否已设置
- 需要重启的配置必须明确标记
- 热更新配置应有反馈

## 调试工具

调试工具面向开发，不应成为普通用户主要入口。

可包含：

- 当前服务状态快照
- 最近 prompt 查看
- LLM 沙箱
- 临时情绪注入
- 临时记忆候选注入
- 客户端日志上报查看

调试工具不应绕过：

- 记忆确认
- 安全边界
- provider 配置保护
- 路由层业务边界

## 后端边界

管理界面通过 `/admin/*` API 访问后端。

设计原则：

- 管理路由只调用服务层
- 不在路由里直接拼 prompt
- 不在路由里直接实现关系更新规则
- 文件操作集中在对应服务中
- destructive 操作需要确认语义

## 设计约束

- 管理界面服务于控制和治理，不服务于沉浸式陪伴
- 所有长期状态必须可见、可改、可删
- 高风险和高隐私能力必须有显式开关
- 调试能力不能绕过安全边界
- 导出和日志默认脱敏
