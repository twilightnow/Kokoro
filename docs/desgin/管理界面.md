# 管理界面

## 1. 目标

管理界面是面向开发和运维的独立后台，用来查看当前 Sidecar 状态、管理角色配置、浏览记忆与日志、调整运行参数，以及执行有限的调试注入。

当前实现优先覆盖两类需求：

1. 日常管理
   角色切换、记忆查看、日志回放、基础配置修改。
2. 调试排障
   当前状态快照、情绪注入、临时事实注入、LLM 沙箱调用。

本文档描述的是“当前仓库内已经落地的版本”，不是早期规划稿。未实现功能会单独列在“暂未覆盖”中，不再混入主流程。

## 2. 当前实现范围

当前管理端由一套独立的 Vue 入口组成：

- 入口页：`frontend/admin.html`
- 启动文件：`frontend/src/admin/main.ts`
- 路由：`frontend/src/admin/router.ts`
- API 客户端：`frontend/src/admin/api.ts`
- 根布局：`frontend/src/admin/AdminApp.vue`

页面共 7 个：

- `Dashboard`
- `Characters`
- `Memories`
- `Logs`
- `EmotionStats`
- `Settings`
- `Debug`

后端接口统一挂在 FastAPI 的 `/admin/*` 下，注册入口在 `src/api/routes/admin/__init__.py`。

## 3. 界面结构

### 3.1 总体布局

管理端采用左侧导航 + 右侧内容区的结构：

- 左侧为固定宽度侧边栏，分为主功能区和开发工具区。
- 右侧为滚动内容区域。
- 底部有一个全局在线状态标记，基于 `/health` 轮询。
- 全局 Toast 由 `AdminApp.vue` 统一提供，子页面通过 `provide/inject` 调用。

### 3.2 导航项

主功能区：

- 状态总览
- 角色管理
- 记忆浏览
- 对话日志
- 情绪统计

开发工具区：

- 配置设置
- 调试工具

## 4. 页面说明

### 4.1 状态总览

对应文件：`frontend/src/admin/views/Dashboard.vue`

用途：

- 展示当前角色 ID / 名称
- 展示当前情绪与剩余持久轮数
- 展示当前会话轮数
- 展示当前记忆摘要数与事实数
- 展示 Sidecar 健康状态
- 展示当前会话累计 token 数

数据来源：

- `GET /state`
- `GET /health`

交互：

- 每 5 秒自动刷新一次
- 支持手动刷新
- 支持把当前情绪直接重置为 `normal`
- 支持跳转到角色管理和记忆浏览页面

说明：

- token 统计来自 `ConversationService.session_token_total`
- 展示值为 `input + output`
- 详细输入输出值通过 hover title 展示

### 4.2 角色管理

对应文件：`frontend/src/admin/views/Characters.vue`

用途：

- 列出 `characters/` 下可用角色
- 标记当前激活角色
- 查看角色 YAML 解析结果
- 编辑并保存 `personality.yaml`
- 手动触发角色重载

数据来源：

- `GET /admin/characters`
- `GET /admin/characters/{character_id}`
- `PUT /admin/characters/{character_id}`
- `POST /admin/characters/{character_id}/reload`

当前交互：

- 左侧点击角色行，右侧加载详情
- 非当前激活角色支持“切换到此”
- 编辑模式直接编辑原始 YAML 文本
- 保存后不会自动生效，需要显式 reload

当前展示字段主要覆盖：

- `name`
- `version`
- `schema_version`
- `personality`
- `behavior_rules`
- `emotion_triggers`
- `forbidden_words`
- `verbal_habits`

### 4.3 记忆浏览

对应文件：`frontend/src/admin/views/Memories.vue`

用途：

- 按角色查看长期事实
- 处理待确认事实冲突
- 查看摘要记忆
- 删除单条摘要
- 清空角色全部记忆文件

数据来源：

- `GET /admin/memories/{character_id}/facts`
- `POST /admin/memories/{character_id}/facts`
- `PUT /admin/memories/{character_id}/facts/{key}`
- `POST /admin/memories/{character_id}/facts/{key}/resolve`
- `DELETE /admin/memories/{character_id}/facts/{key}`
- `GET /admin/memories/{character_id}/summaries`
- `DELETE /admin/memories/{character_id}/summaries/{index}`
- `DELETE /admin/memories/{character_id}`

当前交互：

- 顶部下拉框切换角色
- Facts / Summaries 两个 tab
- 事实支持新增、编辑、删除
- 冲突项支持“采用新值”或“保留旧值”
- 清空全部记忆要求用户输入 `DELETE`

数据文件：

- `data/memories/<character_id>/facts.json`
- `data/memories/<character_id>/summaries.jsonl`

### 4.4 对话日志

对应文件：`frontend/src/admin/views/Logs.vue`

用途：

- 分页列出日志文件
- 查看单个会话 JSONL 的逐轮内容
- 搜索用户输入或回复文本
- 仅筛选命中过禁用词的轮次
- 清空全部日志文件

数据来源：

- `GET /admin/logs`
- `GET /admin/logs/{filename}`
- `DELETE /admin/logs`

当前交互：

- 左侧列表按文件修改时间倒序分页
- 右侧显示逐轮消息、时间、情绪变化、token 用量和命中状态
- 全量清空要求输入 `DELETE`

日志目录：

- `data/logs/*.jsonl`

### 4.5 情绪统计

对应文件：`frontend/src/admin/views/EmotionStats.vue`

用途：

- 查看最近 N 天的情绪时序
- 查看触发词排行

数据来源：

- `GET /admin/stats/emotion?days=N`
- `GET /admin/stats/triggers?top=N&days=N`

当前交互：

- 支持 7 / 30 / 90 天切换
- 前端使用简单 SVG 绘制折线
- 下方表格展示 Top N 触发词

统计规则：

- 情绪时序按 `mood_after` 聚合
- 触发词统计只计算发生情绪变化的记录
- 触发词词表取自当前角色的 `emotion_triggers`
- 后端结果带 60 秒内存缓存

### 4.6 配置设置

对应文件：`frontend/src/admin/views/Settings.vue`

用途：

- 查看当前 `.env` 内容
- 编辑常用 LLM 与记忆相关配置
- 写回 `.env`
- 对可热更新项执行 reload

数据来源：

- `GET /admin/config`
- `PUT /admin/config`
- `POST /admin/config/reload`

当前可编辑的主要字段：

- `LLM_PROVIDER`
- `LLM_MODEL`
- `LLM_MAX_TOKENS`
- `MEMORY_TOKEN_BUDGET`
- `KOKORO_DATA_DIR`
- 供应商对应的 API Key

当前实现约束：

- 敏感 Key 在读取时不回显，只返回是否已设置
- `KOKORO_DATA_DIR` 被标记为“需要重启生效”
- `reload` 只会尝试应用当前已支持的热更新项，不会重建整个服务实例

### 4.7 调试工具

对应文件：`frontend/src/admin/views/Debug.vue`

用途：

- 查看完整内部状态快照
- 查看最近一次发送给 LLM 的 system prompt
- 手动注入情绪
- 注入临时事实
- 执行不落日志的 LLM 沙箱调用

数据来源：

- `GET /admin/debug/state`
- `POST /admin/debug/emotion`
- `GET /admin/debug/inject-fact`
- `POST /admin/debug/inject-fact`
- `DELETE /admin/debug/inject-fact`
- `POST /admin/debug/sandbox`

当前交互：

- 状态支持 2 秒自动刷新
- 情绪注入直接覆盖内存中的 `EmotionState`
- 临时事实只保存在进程内存，不写入磁盘
- 沙箱调用只请求 LLM，不修改主会话状态，不写会话日志

## 5. 后端实现

### 5.1 路由注册

主应用在 `src/api/app.py` 中注册：

- 聊天相关：`chat` / `state` / `stream`
- 管理相关：`admin_router`

管理子模块如下：

- `src/api/routes/admin/characters.py`
- `src/api/routes/admin/memories.py`
- `src/api/routes/admin/logs.py`
- `src/api/routes/admin/stats.py`
- `src/api/routes/admin/debug.py`
- `src/api/routes/admin/config_mgr.py`

### 5.2 服务共享与角色切换

为了避免路由层与 `app.py` 形成循环依赖，当前通过 `src/api/service_registry.py` 保存全局 `ConversationService` 实例。

角色切换逻辑：

1. 调用旧实例的 `_on_session_end()` 持久化当前会话摘要。
2. 基于新角色 `personality.yaml` 创建新的 `ConversationService`。
3. 用新实例替换注册表中的全局引用。

### 5.3 状态扩展

为了给管理界面提供更多可观测性，`ConversationService` 增加了以下状态输出：

- `last_system_prompt`
- `session_token_total`
- `working_memory_message_count`
- `working_memory_truncation_count`

`GET /state` 额外返回：

- `display`
- `session_token_total`

## 6. 数据与安全边界

### 6.1 文件读写范围

当前管理端会直接操作以下本地文件：

- `characters/*/personality.yaml`
- `.env`
- `data/memories/*/facts.json`
- `data/memories/*/summaries.jsonl`
- `data/logs/*.jsonl`

### 6.2 已做的基本保护

- 日志读取对 `filename` 做了基础路径穿越拦截
- Live2D 资源访问会校验路径必须位于 `characters/` 根目录之内
- API Key 在配置读取接口中不回显真实值
- 清空类操作要求显式确认

### 6.3 当前设计假设

- `/admin/*` 仅面向本机 `127.0.0.1:18765`
- 前端通过固定 base URL 访问 Sidecar
- 当前没有额外鉴权层
