# 管理界面

## 目标
定义 Kokoro 当前管理界面的真实能力边界: 观测运行状态、编辑角色与配置、管理记忆和关系、控制主动陪伴、管理感知隐私、查看日志与统计、执行调试操作。

## 范围
### 包含
- `frontend/src/admin/` 当前已有的 hash 路由和视图。
- `/state`、`/health` 与 `/admin/*` 的调用映射。
- 设置页、交互页、主动陪伴页、感知隐私页、提醒页与调试页的真实行为边界。

### 不包含
- 不写未实现的 STT 管理或多用户管理功能。
- 不重复描述主窗口布局和角色显示实现。

## 输入 / 输出
### 输入
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `/state` | JSON | Dashboard 时是 | 当前角色、情绪、记忆、关系和 token 总览。 |
| `/health` | JSON | Dashboard 时是 | sidecar、LLM、资源、TTS 健康状态。 |
| `/admin/characters/*` | JSON | Characters 页时是 | 角色清单、详情、原始 YAML、资源验证结果。 |
| `/admin/memories/*` | JSON | Memories 页时是 | 记录仓库视图下的 facts、candidate、summaries、导出与清理。 |
| `/admin/relationship/*` | JSON | Relationship 页时是 | 关系状态读取、编辑、重置和表达边界摘要。 |
| `/admin/reminders/*` | JSON | Reminders 页时是 | reminder 的创建、列表、更新、完成、改期和删除。 |
| `/admin/proactive/*` | JSON | Proactive 页时是 | 主动陪伴设置、状态、日志、测试与反馈。 |
| `/admin/perception/*` | JSON | Perception 页时是 | 感知隐私设置、采集状态和脱敏审计摘要。 |
| `/admin/logs/*` | JSON | Logs 页时是 | 对话日志清单与明细。 |
| `/admin/stats/*` | JSON | Stats 页时是 | 情绪统计与触发统计。 |
| `/admin/debug/*` | JSON | Debug 页时是 | 工作记忆、结构化情绪快照、当前情绪片段、最近 5 条情绪事件、最近安全事件、最近结束片段、注入情绪、临时事实、sandbox、客户端日志和 session flush 等调试能力。 |
| `/admin/config/*` | JSON | Settings / Interaction 时是 | `.env` 配置读写、热更新、LLM 测试。 |
| `localStorage` | key-value | Interaction 时是 | 窗口和角色展示偏好、语音输入来源、TTS 开关。 |

### 输出
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 视图渲染 | DOM | 是 | Dashboard、Characters、Memories、Relationship、Proactive、Perception、Reminders、Logs、Stats、Interaction、Debug、Settings。 |
| `/admin/*` 请求 | HTTP | 是 | 用户操作触发的管理请求。 |
| `localStorage` 更新 | key-value | 否 | 交互设置页写回本地偏好。 |
| Tauri 命令 | invoke | 否 | Interaction 页尝试同步穿透、置顶、开机启动。 |

## 流程
1. `frontend/src/admin/main.ts` 启动 admin 应用，`router.ts` 注册十二条页面路由。
2. `Dashboard` 每 5 秒轮询 `/state`、`/health` 与部分配置，展示角色、情绪、记忆、关系、资源和 LLM/TTS 状态；当 TTS 被显式关闭时显示 `disabled` 和禁用说明。
3. `Characters` 读取角色列表、详情、manifest 验证结果，并支持更新 YAML、重载角色、设置默认启动角色。
4. `Memories` 管理事实、候选确认、摘要、冲突解决、导出和清空；当前前端仍通过 `pending_confirm`、`pending_value` 等兼容字段感知候选状态。
5. `Relationship` 读取并人工编辑 `relationship_type`、`preferred_addressing`、`boundaries_summary`，展示当前关系类型对应的表达边界摘要，也可重置默认值。
6. `Proactive` 独立管理主动陪伴总开关、模式、勿扰时间、场景开关、游戏场景介入等级，并展示今日状态与最近主动日志。
7. `Perception` 独立管理隐私过滤开关、应用/标题黑名单、敏感脱敏规则、最大标题长度、脱敏审计开关、隐私勿扰应用/标题/会议/全屏规则，并展示最近过滤后的审计摘要。
8. `Reminders` 独立管理本地 reminder 内容: 选择角色、创建、编辑、完成、顺延 30 分钟、删除，并可切换是否显示已完成项。
9. `Logs` 与 `Stats` 提供日志浏览、情绪统计和触发统计。
10. `Debug` 查看工作记忆、token 历史，以及当前情绪的触发词、原因、来源、强度、恢复速率、预计剩余轮数、当前片段、最近 5 条事件、最近安全事件和最近结束片段；也可注入情绪、注入临时事实、查看客户端日志并运行 sandbox；桌面端退出前复用 `/admin/debug/flush-session` 触发一次增量记忆结算。
11. `Settings` 读写 `.env`、测试 LLM、热更新可热更新的配置、导出诊断；TTS 开关通过 `TTS_PROVIDER=edge-tts|disabled` 显式持久化；感知采集总开关通过 `KOKORO_ENABLE_PERCEPTION=0|1` 持久化并提示重启生效。
12. `Interaction` 只同步主窗口本地偏好和系统开机项，并尝试即时调用 Tauri 命令；其中 TTS 开关与主窗口静音按钮共享同一 `localStorage` 键。

## 约束与规则
- 管理界面允许在浏览器和 Tauri admin 窗口中运行，不能假设 Tauri API 一定存在。
- `/admin/config` 的 `updates` 必须发送字符串字典；数值输入在请求前必须转成字符串。
- Settings 页对敏感 key 只允许替换或清空，不回显明文。
- Interaction 页保存的窗口缩放、角色缩放和角色偏移只存本地，不进入 `.env`。
- 主动陪伴设置必须通过 `/admin/proactive/settings` 写入本地持久化文件；不再要求用户通过 `.env` 手动切换主动性。
- 感知采集总开关必须通过 Settings 页写入 `.env` 的 `KOKORO_ENABLE_PERCEPTION`，并提示需要重启 sidecar 生效。
- 隐私过滤、黑名单、脱敏规则、审计和隐私勿扰规则必须通过 `/admin/perception/settings` 写入 `data/runtime/perception/privacy_settings.json`。
- Perception 页展示的审计列表只能来自 `/admin/perception/audit` 的脱敏摘要，不能展示或缓存原始窗口标题。
- 当 Perception 页关闭隐私过滤时，后端不得继续写入感知审计记录。
- reminder 内容必须通过 `/admin/reminders/*` 写入 `data/runtime/reminders/<character_id>.json`；`Proactive` 页不得承担 reminder 文本编辑。
- `/admin/debug/emotion` 只覆盖当前 `ConversationService` 内存中的 EmotionTimeline / EmotionState，不写日志也不写磁盘。
- Debug 页中的情绪事件列表显示最近 5 条触发事件；timeline 片段列表显示当前片段和最近结束片段，不写磁盘。
- Debug 页中的安全事件列表只能显示风险级别、动作、规则名、关系类型和轮次，不显示完整原文。
- Relationship 页必须展示 `boundary_policy_summary`，且该文本随 `relationship_type` 改变。
- 若未来接入向量检索，Memories 页仍以记录仓库为主视图，向量召回结果只能作为辅助检索，不直接替代管理对象。
- 当 `.env` 中 `TTS_PROVIDER=disabled` 时，Dashboard 必须显示已禁用状态；主窗口本地静音与后端禁用都不得显示“不支持的 provider”。

## 验收标准
- 打开 admin 首页；能看到 Dashboard、Characters、Memories、Relationship、Proactive、Perception、Reminders、Logs、Stats、Interaction、Debug、Settings 十二个路由入口。
- Dashboard 正常显示 `/state` 和 `/health` 中的角色、LLM、资源与 TTS 状态。
- Settings 保存配置；后端返回 `restart_required` 时页面显示重启提醒。
- 在 Settings 中关闭 TTS 后刷新 Dashboard；TTS 状态显示 `disabled`，副标题显示“已禁用”而不是 provider 错误。
- Interaction 保存主窗口偏好；对应 `localStorage` 键被更新，且即使 Tauri 命令失败也不会阻断页面。
- Proactive 保存设置并发送测试动作；状态卡片更新且最近日志出现一条新的主动记录。
- Perception 保存标题黑名单并刷新页面；黑名单仍出现在设置文本框中。
- Perception 审计列表返回一条含 `[已脱敏]` 的摘要；页面显示脱敏标题和脱敏规则名，不显示原始敏感片段。
- Reminders 创建一条一次性 reminder 并点击“稍后 30 分钟”；列表中的 `due_at` 被推进，且无需修改 `Proactive` 页设置。
- 用户输入命中情绪关键词后打开 Debug；页面显示触发词、原因、来源、强度、预计剩余轮数和最近 5 条事件中的最新记录。
- 用户输入命中情绪关键词后打开 Debug；页面显示当前片段的已持续轮数、恢复速率和可选 TTS 调整，并能看到历史片段的结束原因。
- Debug 注入情绪后刷新 Dashboard；当前情绪发生变化。
- 触发一次危机短路后打开 Debug；最近安全事件显示 `level=crisis` 和规则名，不显示原始输入。
- Relationship 页切换关系类型并保存；边界摘要刷新为新关系类型对应文本。
