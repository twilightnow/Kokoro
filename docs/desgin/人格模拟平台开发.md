# Kokoro 当前设计总览

这份文档只描述当前代码已经落地的结构，以及短期内仍然成立的模块边界。

项目当前是一个本地运行的桌面人格伴侣原型，分成三部分：

- Python CLI / FastAPI sidecar：负责对话主流程、人格、记忆、感知、日志
- Tauri 壳：负责桌面窗口、托盘、窗口行为和前端事件桥接
- Vue 前端：负责展示、输入和 WebSocket 通信

当前目标不是堆功能，而是先把三件事做稳：

- 角色说话风格稳定
- 模块之间简单解耦
- 桌面形态可持续扩展

## 当前运行结构

```text
CLI / Tauri + Vue
        |
        v
FastAPI sidecar / ConversationService
        |
        +-- personality
        +-- memory
        +-- perception
        +-- capability.llm
        +-- logger
```

对应目录：

- `main.py`：CLI 入口
- `src/api`：sidecar API
- `src/application`：应用编排层
- `src/personality`：角色配置、情绪状态、prompt 拼装
- `src/memory`：工作记忆、摘要记忆、长期事实记忆
- `src/perception`：窗口/活跃状态采集与主动触发
- `src/capability`：LLM 抽象和 provider 适配
- `src-tauri`：桌面壳与托盘命令
- `frontend/src`：桌面前端界面

## 分层原则

- UI 层只做显示、输入和桌面行为，不做人格判断
- API 层只暴露协议，不承载核心业务
- `ConversationService` 只做编排，不直接写死人格规则
- 人格层统一生成 system prompt
- 记忆层和感知层只输出结构化上下文
- 能力层只提供 LLM 调用能力，不决定什么时候调用

这套边界已经在代码里基本成立，后续扩展优先沿着这条线继续做，不引入跨层捷径。

## 当前已实现能力

- CLI 对话与日志回放
- FastAPI HTTP + WebSocket sidecar
- 角色 YAML 加载与校验
- 轻量情绪状态机
- 工作记忆 + 摘要记忆 + 长期事实记忆
- 基于窗口标题、活跃状态、时间段的轻感知
- Tauri 透明悬浮窗、托盘、边缘吸附、点击穿透
- 多角色目录加载与运行时切换

## 当前未落地或仍属预留

- Live2D / VRM 渲染
- TTS / 语音输入
- OCR / 截图视觉理解
- 台词库 RAG
- 角色包导入导出
- sidecar 由 Tauri 自动拉起

这些能力可以扩，但不应反向污染当前简单结构。

## 当前开发判断

短期内把下面三层继续保持稳定最重要：

1. `ConversationService` 继续做薄编排层
2. `PromptContext` / `MemoryContext` / `PerceptionContext` 继续做跨层数据契约
3. 前端继续只消费 `/state`、`/health`、`/stream` 和角色切换接口

如果后面增加新能力，优先新增模块和契约，不要把逻辑直接塞回 UI、路由或 LLM client。
