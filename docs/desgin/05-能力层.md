# 能力层

## 目标
定义 Kokoro 当前对外部能力的适配层边界: LLM 对话能力与 TTS 合成能力。

## 范围
### 包含
- `src/capability/llm.py` 的 provider 解析、调用结果结构和降级行为。
- `src/capability/tts.py` 的 Edge TTS 适配。
- `LLMResult`、`TTSResult` 与流式调用退化策略。

### 不包含
- 不把 STT、Embedding、视觉理解、RAG 写成现状。
- 不定义角色、记忆、关系或前端渲染逻辑。

## 输入 / 输出
### 输入
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `LLM_PROVIDER` | string | 否 | 可为 `claude-cli`、`gemini-cli`、`codex-cli`、`anthropic`、`openai`、`gemini`、`deepseek`、`openrouter`、`copilot`。 |
| `LLM_MODEL` | string | 否 | 覆盖 provider 默认模型。 |
| `system_prompt` | string | 是 | LLM 对话调用的 system prompt。 |
| `messages` | `list[{role, content}]` | 是 | LLM 对话消息历史。 |
| `TTS_PROVIDER` | string | 否 | 当前支持 `edge-tts`；值为 `disabled` 时显式禁用 TTS。 |
| `text` `voice?` `rate?` `volume?` | string | TTS 时是 | TTS 合成输入。 |

### 输出
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `LLMResult` | dataclass | 是 | 包含 `text`、`model`、`provider`、`input_tokens`、`output_tokens`、`finish_reason`。 |
| `stream_chat()` token 序列 | generator | 否 | 当前 provider 若无原生流式，则退化为一次性结果。 |
| `TTSResult` | dataclass | 是 | 包含 `audio_bytes`、`content_type`、`voice`。 |

## 流程
1. `create_llm_client()` 从显式参数或 `.env` 解析 provider 名称、模型和凭证。
2. provider 名称支持别名归一化，并在缺少配置或冲突时抛出 `EnvironmentError`。
3. LLM client 执行 `chat()`，返回 `LLMResult`；`stream_chat()` 默认退化为完整文本再一次性回传。
4. `ConversationService` 和 `/stream` 消费 `LLMResult` 与 token 序列。
5. `create_tts_client()` 解析 `TTS_PROVIDER`；值为 `disabled` 时返回“已禁用”状态，不再按无效 provider 处理。
6. `EdgeTTSClient.synthesize()` 调用 `edge_tts.Communicate.stream()`，拼接音频字节后返回 `TTSResult`。
7. 情绪联动的 `rate`、`volume` 由前端或其他调用方根据 `emotion` 摘要显式传入 `/tts`；能力层不读取全局情绪状态。

## 约束与规则
- LLM provider 不可用时，聊天主链路返回错误，不自动切换到其他 provider。
- CLI provider 和 API provider 共用统一 `LLMClient` 抽象。
- `copilot` provider 读取 `GITHUB_TOKEN`、`GH_TOKEN` 或 `COPILOT_GITHUB_TOKEN`。
- 当前 TTS 只有 `edge-tts` 合成实现；`disabled` 只表示显式关闭语音输出，不属于无效 provider。
- `TTS_PROVIDER=disabled` 时，`/health` 必须返回 `tts.status=disabled`，`/tts` 必须返回“已禁用”而不是“不支持的 provider”。
- 能力层必须保持 TTS 无状态；同一段文本的语速和音量只由本次请求参数决定。
- 当前没有 STT、Embedding、视觉理解或工具调用编排层。
- `[待定:是否引入 STT provider 尚未定义]`。

## 验收标准
- 未配置任何 LLM provider时启动服务；聊天请求返回 provider 未配置相关错误。
- 配置 `LLM_PROVIDER=deepseek` 与合法 key；聊天调用返回 `LLMResult.provider=deepseek`。
- 调用 `/stream`；即使底层 provider 不支持原生流式，也能以 `done` 帧结束。
- 配置 `TTS_PROVIDER=edge-tts` 后请求 `/tts`；返回音频字节流。
- 配置 `TTS_PROVIDER=disabled` 后请求 `/health`；返回 `tts.status=disabled` 和 `message=TTS 已禁用`。
- 配置 `TTS_PROVIDER=disabled` 后请求 `/tts`；返回“已禁用”错误而不是“不支持的 provider”。
- 配置不支持的 `TTS_PROVIDER`；请求 `/tts` 返回错误而不是进程崩溃。
