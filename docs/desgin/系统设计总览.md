# 系统设计总览

Kokoro 是本地优先的桌面 AI 伴侣系统。它不是单纯的聊天 UI，也不是把 LLM 包一层角色 prompt，而是由人格、记忆、情绪、感知、表现和安全边界共同组成的长期陪伴运行时。

当前代码已经具备桌面悬浮窗、FastAPI sidecar、角色配置、轻量情绪、三级记忆、轻感知、Live2D/3D 展示和管理后台。后续设计目标是在不破坏这些边界的前提下，补足关系状态、生活事件、承诺提醒、主动调度和安全边界。

## 文档职责边界

`docs/desgin/` 只维护设计，不维护路线图。

本目录负责：

- 系统架构
- 模块职责
- 跨层数据契约
- 状态与存储边界
- 隐私和安全原则
- 后续扩展时不能破坏的约束

本目录不负责：

- 阶段优先级
- 任务清单
- 完成状态
- v1 范围
- 验收标准

这些内容统一放在 `docs/tasks/devRoute/`。设计文档可以说明某个能力应如何被设计，但不记录“什么时候做”和“是否完成”。

## 系统分层

```text
CLI / Tauri + Vue
        |
        v
FastAPI sidecar / IPC
        |
        v
Application Layer
        |
        +-- personality
        +-- memory
        +-- perception
        +-- relationship
        +-- capability
        +-- logger
```

### UI / Desktop

负责桌面存在感、输入输出、角色展示、窗口行为、设置入口和管理界面。

UI 不负责：

- 拼 prompt
- 判断人格
- 直接读写记忆文件
- 直接决定关系数值
- 直接调用 provider key

### API / IPC

负责把本地应用能力暴露给前端。IPC 只定义协议，不承载业务决策。

路由层不应直接操作人格、记忆和感知内部文件。所有核心业务应通过 application 层服务完成。

### Application

负责把单轮对话、长期运行时、主动行为、提醒、关系状态和日志编排起来。

当前核心是 `ConversationService`。后续长期伴侣能力应新增独立服务，例如 `CompanionRuntime`、`RelationshipService`、`ProactiveScheduler`、`RoutineReminderService`，而不是继续把所有逻辑塞进 `ConversationService`。

### Personality

负责角色配置、情绪状态和 prompt 统一拼装。

人格层是表达统一出口。记忆、感知、关系和安全策略都应作为结构化输入进入人格层，而不是绕过人格层直接改写回复。

### Memory

负责把历史信息转成可治理、可裁剪、可解释的上下文。

记忆不是越多越好。系统应区分事实、摘要、偏好、边界、生活事件和承诺提醒，避免所有长期信息都混进一个事实表。

### Perception

负责采集有限、本地、可关闭的环境信号，并输出结构化场景。

感知层只描述场景，不生成台词，不决定人格，不直接触发 UI 打扰。是否主动介入应由主动调度层判断。

### Capability

负责封装外部或本地能力，例如 LLM、TTS、STT、Embedding、视觉理解等。

能力层只提供能力，不决定什么时候调用、不持有人格规则、不写业务状态。

## 核心数据契约

跨层通信应优先使用稳定上下文对象，而不是散乱字段：

- `PromptContext`：人格层输入，汇总角色、情绪、记忆、感知、关系和安全提示
- `MemoryContext`：记忆层输出，提供已裁剪、可注入的记忆摘要
- `PerceptionContext`：感知层输出，描述当前窗口、时间、活跃状态和高层活动
- `RelationshipState`：关系层状态，描述亲密度、信任度、熟悉度、边界偏好和依赖风险
- `EmotionTimeline`：情绪连续状态，描述当前情绪、强度、原因、来源和恢复趋势
- `DisplayState`：表现层输入，描述情绪、说话、思考、待机、注意力和一次性反应

新增能力优先扩展这些契约或新增专用上下文对象，不应让上层直接读取底层存储。

## 长期伴侣设计方向

“像伴侣”在系统设计里不等于伪装成人类，而是让系统具备长期一致性、可治理记忆、合适主动性和明确边界。

长期伴侣体验依赖这些能力：

- 稳定人格：角色核心设定不随单轮对话漂移
- 关系状态：关系随互动缓慢变化，但可查看、可重置
- 生活记忆：能记住共同经历、近期目标、压力源和重要日期
- 承诺系统：能结构化保存提醒、约定和待跟进事项
- 主动调度：知道什么时候说话、什么时候只表现、什么时候沉默
- 情绪连续性：情绪有原因、强度、持续时间和恢复过程
- 表现联动：文字、语音、表情、动作和桌面状态一致
- 安全边界：不进行身份伪装、依赖绑定和现实关系隔离

## 设计约束

- 本地优先，敏感数据默认留在用户机器
- 所有长期状态必须可查看、可修改、可删除
- 角色包和用户数据必须分离
- 主动行为必须可关闭、可降频、可解释
- 感知能力必须可配置、可审计、可脱敏
- 安全策略不能只依赖 prompt
- 新增能力必须能降级，不应阻断核心对话
- 路线图只链接设计，不复制完整设计说明
