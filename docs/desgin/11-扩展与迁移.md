# 扩展与迁移

## 目标
定义 Kokoro 当前角色内容扩展和兼容迁移的真实边界: 角色目录结构、配置入口、显示资源入口，以及当前尚未实现的自动迁移边界。

## 范围
### 包含
- 角色目录 `characters/<character_id>/` 的当前入口文件约定。
- `personality.yaml` 与可选 `manifest.yaml` 的职责边界。
- 默认启动角色、角色切换、角色资源查找的当前行为。

### 不包含
- 不把插件系统、角色包安装器、自动 schema 迁移工具写成现状。
- 不定义远程下载、市场、签名校验或跨仓库分发机制。

## 输入 / 输出
### 输入
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `characters/<character_id>/personality.yaml` | file | 是 | 角色人格配置入口。 |
| `characters/<character_id>/manifest.yaml` | file | 否 | 角色显示资源入口。 |
| `KOKORO_DEFAULT_CHARACTER` | string | 否 | 默认启动角色 ID。 |
| `/switch-character?name=...` | query | 否 | 运行时切换角色。 |
| `/admin/characters/{character_id}` PUT body | `{raw_yaml: string}` | 否 | 角色 YAML 编辑输入。 |

### 输出
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `CharacterConfig` | dataclass | 是 | 人格配置加载结果。 |
| `display` 配置 | JSON | 否 | 从 `manifest.yaml` 派生的显示配置。 |
| 默认角色解析结果 | path / id | 否 | CLI 与 sidecar 启动时使用的角色入口。 |

## 流程
1. 角色目录以文件夹名作为 `character_id`。
2. 启动时通过 `resolve_default_character_path()` 或 `KOKORO_DEFAULT_CHARACTER` 选择初始角色。
3. 人格层总是从 `personality.yaml` 读取角色配置。
4. 显示层若存在 `manifest.yaml`，则从中解析 `display.mode` 及资源入口。
5. 运行时角色切换通过 `/switch-character` 替换当前服务实例。
6. 管理界面编辑角色时直接提交 YAML 文本，再由后端写回文件并触发重载。

## 约束与规则
- `character_id` 由目录名决定，当前没有额外 manifest 级唯一标识。
- `personality.yaml` 是必需文件；`manifest.yaml` 是可选文件。
- 用户本地偏好、关系状态、记忆和日志存放在 `data/` 下，不写回 `characters/`。
- 当前没有自动 schema 迁移器；字段变化需通过文档和人工修正处理。
- 当前没有角色包导入导出 API。
- `[待定:角色 schema 自动迁移策略尚未定义]`。

## 验收标准
- 新建 `characters/<id>/personality.yaml` 且字段合法；服务可加载该角色。
- 缺少 `personality.yaml`；启动或切换角色时报错。
- 新建合法 `manifest.yaml`；`/state.display` 返回对应显示模式配置。
- 通过 admin 修改角色 YAML 并重载；当前角色配置被更新。
