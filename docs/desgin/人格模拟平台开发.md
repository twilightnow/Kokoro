# Kokoro — 桌面 AI 人格伴侣平台 · 总览

以 Live2D 立绘为交互界面的桌面 AI Agent，像 VTuber 一样呈现，具备屏幕感知、长期记忆、主动介入能力。角色通过配置文件驱动，支持多角色扩展。

平台核心目标只有一个：先做出“稳定像她”的角色对话体验。记忆、感知、Live2D、TTS 都是增强项，不能反过来压垮主体验。

产品定位进一步收敛为：**情境陪跑型桌面伴侣**。

她不是通用办公 Agent，也不是纯展示型桌宠，而是在用户工作、发呆、深夜熬夜、轻度娱乐等场景中，以克制但有个性的方式陪着用户。

---

## 设计原则

- **角色优先** — 所有架构决策以"她像她"为最高标准
- **本地优先** — 数据全部存本地
- **成本可控** — 感知层分级，避免无效API调用
- **可扩展** — 角色以独立包形式安装，支持创意工坊分享
- **可迁移** — 用户数据集中单目录，备份即迁移
- **先稳后多** — 首版先验证人格对话稳定性，再逐步加记忆、感知、表现层
- **低打扰常驻** — 默认像桌面陪伴者，而不是持续打断用户的助手
- **场景驱动开口** — 她的价值来自“在合适时机出现”，而不是“随时抢着说话”

## 架构分层

| 层 | 职责 | 详细 |
|----|------|------|
| UI层 | Tauri壳 + 角色表现层 + 系统托盘 | [[UI层]] |
| 人格层 | 角色配置 + 情绪状态机 + 台词RAG | [[人格层]] |
| 记忆层 | 对话历史 + 长期记忆 + 关系状态 | [[记忆层]] |
| 感知层 | 屏幕感知 + 窗口监听 + 主动介入 | [[感知层]] |
| 能力层 | LLM + 视觉API + TTS | [[能力层]] |

完整桌面版运行时：Tauri → IPC → Python FastAPI sidecar，用户无需单独安装Python。

补充说明：
- Demo 阶段仅使用 CLI，不引入 Tauri 和 FastAPI sidecar
- FastAPI sidecar 属于桌面版运行时组件，在后续表现层阶段引入

约束：
- UI层只负责展示和交互，不做人格判断
- 人格层统一决定最终输出
- 记忆层、感知层、能力层只向人格层提供输入，不直接决定回复

## 使用方向

主方向：**情境陪跑型桌面伴侣**

适用场景：

- 用户工作中，需要低频但有存在感的陪伴
- 用户深夜仍在活跃，需要轻量提醒或情绪反馈
- 用户短暂发呆、切窗口、摸鱼时，希望有角色化互动
- 用户主动打开聊天时，希望她保持稳定人格，不像普通助手

不作为首版主方向的内容：

- 通用办公 Agent
- 重任务自动执行
- 重视觉监控型助手
- 工坊平台优先驱动的角色分发系统

一句话描述：

**像桌宠一样常驻，像角色一样说话，像陪伴者一样懂一点场景。**

## 开发路线图

- **Phase 1** — 跑通人格对话闭环：角色配置 + 情绪状态 + 基础对话 + 日志记录
- **Phase 2** — 增加轻记忆与轻感知：会话摘要、窗口标题、活跃状态、谨慎主动介入
- **Phase 3** — 增强表现与扩展：Tauri、Live2D、TTS、多角色、角色包导入

## 注意事项

角色素材（Live2D模型、语音）需确认各自来源的使用条款。屏幕感知数据仅本地处理，不上传。

首版验收标准以“人格是否稳定”为主，不以功能数量为主。

使用体验上，首版优先验证三点：

- 她是否足够像她
- 她是否足够不烦人
- 她是否在少数高价值场景下出现得合理
