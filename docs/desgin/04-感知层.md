# 感知层

## 目标
定义 Kokoro 当前本地感知、隐私过滤与主动介入的真实实现: 采集窗口和输入活动，先生成脱敏后的 `PerceptionContext`，再在后台 runtime 中产出结构化主动信号。

## 范围
### 包含
- `InputTracker`、`WindowMonitor`、`PerceptionCollector`、`ProactiveSignalDetector` 与 `CompanionRuntime` 的当前职责。
- `PrivacySettings`、`PrivacyFilter`、`PerceptionAuditRepository` 的过滤、勿扰和脱敏审计边界。
- `late_night`、`long_work`、`idle_return`、`window_switch`、`gaming` 五类感知信号与 `reminder` 运行时信号的接入边界。
- 主动日志、抑制决策和初始化失败时的降级行为。

### 不包含
- 不定义复杂活动分类器、截图、OCR 或屏幕内容理解。
- 不定义外部视觉模型、系统截图或语音输入识别。

## 输入 / 输出
### 输入
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `KOKORO_ENABLE_PERCEPTION` | string flag | 否 | sidecar 或 CLI 是否启用感知层。 |
| 键鼠活动 | runtime event | 是 | 由 `InputTracker` 计算 `idle_seconds`。 |
| 当前窗口标题 | string | 是 | 由 `WindowMonitor.collect()` 提供。 |
| 当前应用名 | string | 否 | 由 `WindowMonitor.current_app_name()` 提供；当前实现失败时为空字符串。 |
| 当前时间 | datetime | 是 | 用于 `hour`、`is_late_night` 和 `time_of_day`。 |
| `data/runtime/reminders/<character_id>.json` | file | 否 | `RoutineReminderService` 读取 reminder 到期状态。 |
| `data/runtime/proactive/settings.json` | file | 否 | 控制 DND、每日上限、场景开关和游戏场景介入等级。 |
| `data/runtime/perception/privacy_settings.json` | file | 否 | 控制隐私过滤、黑名单、脱敏、审计和隐私勿扰规则。 |

### 输出
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `PerceptionContext` | dataclass | 是 | 包含过滤后的 `active_window_title`、`active_app_name`、`is_user_active`、`hour`、`idle_seconds`、`switches_per_minute`、`is_gaming`、`is_fullscreen`、`blocked_reason`、`dnd_reason`、`redactions`。 |
| `ProactiveSignal` | dataclass | 否 | 包含 `scene`、`reason`、`trigger_name`、`priority` 和可选 metadata。 |
| `data/runtime/proactive/events.jsonl` | file | 否 | sent 与 suppressed 的主动决策日志。 |
| `data/runtime/perception/audit.jsonl` | file | 否 | 只保存脱敏后的感知摘要，不保存原始窗口标题。 |

## 流程
1. sidecar lifespan 启动时，`CompanionRuntime` 初始化 `InputTracker`、`WindowMonitor`、`PerceptionCollector` 和 `ProactiveSignalDetector`。
2. runtime 每轮先查询 `RoutineReminderService.due(now)`，把到期 reminder 转换为 `scene=reminder` 的 `ProactiveSignal`。
3. 若 collector 可用，runtime 再调用 `PerceptionCollector.collect()`，采集当前窗口标题、应用名、空闲秒数、窗口切换频率和是否在游戏。
4. collector 创建原始上下文后立即读取 `PrivacySettings` 并调用 `PrivacyFilter.apply()`；黑名单命中时清空标题，敏感规则命中时替换为 `[已脱敏]`，超长标题按 `max_title_length` 截断。
5. 隐私过滤和审计同时开启时，collector 只把过滤后的摘要写入 `data/runtime/perception/audit.jsonl`；隐私过滤关闭时不得写入感知审计。
6. 每次采集后，runtime 将最近过滤后的 `PerceptionContext` 写回 `ConversationService`，供后续普通对话 prompt 注入；collector 不可用时写回 `None`。
7. `ProactiveSignalDetector` 维护持续活跃计时和前一帧感知快照，按优先级产出 `long_work`、`late_night`、`idle_return`、`window_switch`、`gaming` 信号。
8. `ProactiveScheduler` 结合主动设置、本日 sent 次数、最小间隔和 `PerceptionContext.dnd_reason`，统一评估 reminder 与 perception-derived 信号，决定发送、降级还是抑制。
9. `dnd_reason` 非空时，主动策略返回 `suppressed_by=privacy_dnd:<reason>`，并且不广播主动气泡。
10. 允许发送时，主动行为由 runtime 走模板或 LLM 生成后广播；抑制时也可记录 `suppressed_by` 以便后续调参。
11. collector 初始化失败时，只禁用 perception-derived 主动行为；reminder 仍可继续经过策略层与广播链路。
12. sidecar 关闭时停止 `InputTracker` 并结束 runtime loop。

## 约束与规则
- 感知层只输出结构化上下文和事件，不直接生成业务回复或修改情绪状态。
- 当前 `is_user_active` 的阈值是 `idle_seconds < 300`。
- 触发阈值固定为: idle return 依赖上一个长空闲周期至少 1800 秒、window switch 10 次/分钟、long work 3600 秒。
- 原始窗口标题只能存在于 collector 调用栈内；进入 prompt、主动策略、主动日志和感知审计前必须使用过滤后的 `PerceptionContext`。
- 应用黑名单或标题黑名单命中时，`active_window_title` 必须为空字符串，`blocked_reason` 必须非空，`dnd_reason` 必须为 `privacy_blocked`。
- 脱敏规则默认覆盖邮箱、key/token/secret/password 片段、带 query 的 URL、12 位以上数字串和 Windows 用户目录路径。
- 感知审计只能写入过滤后的窗口标题、应用名、时间段、活跃状态、空闲秒数、切换频率、游戏/全屏标记、屏蔽原因、勿扰原因和脱敏规则名。
- 当 `PrivacySettings.enabled=false` 时，collector 不写入感知审计，避免 `audit_enabled=true` 单独造成原始标题落盘。
- `dnd_meeting_patterns` 默认命中会议关键词时只抑制主动陪伴，不阻止本地过滤后上下文进入普通聊天 prompt。
- sidecar 的主动介入由后台 runtime loop 驱动；CLI 的 `--perception` 只按轮采集最新感知快照注入 prompt，不再承担轮间主动开口。
- reminder 信号不依赖感知采集器；其唯一外部输入是本地 reminder 仓库和当前时间。

## 验收标准
- 启用 `KOKORO_ENABLE_PERCEPTION=1` 启动服务；感知模块初始化成功时不影响正常聊天。
- 模拟一个长时间空闲后重新活跃的快照切换；后台 detector 产生 `idle_return` 信号。
- 在深夜时段且用户活跃时运行 runtime 一次；存在 `late_night` sent 或 suppressed 记录。
- 在勿扰时间内满足主动条件；日志记录 `decision=suppressed` 且 `suppressed_by=dnd`。
- 感知初始化失败；服务仍然可以通过 `/chat` 或 CLI 正常回复，且到期 reminder 仍可产生 `scene=reminder` 的 proactive 记录。
- 配置标题黑名单 `Secret` 后采集窗口标题 `Secret Plan`；prompt 中不出现 `Secret Plan`，审计记录 `blocked_reason=blocked:title`。
- 采集窗口标题包含邮箱地址；`data/runtime/perception/audit.jsonl` 中只出现 `[已脱敏]`，不出现原始邮箱。
- 配置勿扰标题 `Meeting` 后采集窗口标题 `Weekly Meeting` 并满足主动条件；`data/runtime/proactive/events.jsonl` 记录 `decision=suppressed` 且 `suppressed_by=privacy_dnd:title` 或 `privacy_dnd:meeting`。
