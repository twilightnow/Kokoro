# 长期伴侣运行时

## 目标
定义 Kokoro 当前与长期陪伴最相关的运行时实现: 关系状态持久化、后台 `CompanionRuntime`、主动陪伴设置与主动行为日志。

## 范围
### 包含
- `RelationshipState`、`RelationshipService` 的当前字段、更新逻辑和持久化位置。
- `ConversationService` 中关系摘要注入和互动记录的真实时机。
- `CompanionRuntime` 的后台循环、主动设置读取、决策日志与广播行为。
- `RoutineReminderService` 的本地 reminder 数据模型、到期推进和反馈回写规则。
- admin 对关系状态、主动陪伴设置和 reminder 的管理能力。

### 不包含
- 不把跨会话情绪记忆、复杂承诺图谱或系统日历集成写成已实现。
- 不定义复杂承诺图谱、系统日历集成或自动抽取 reminder 意图。

## 输入 / 输出
### 输入
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `character_id` | string | 是 | 关系状态按角色隔离。 |
| `user_input` | string | 互动记录时是 | 本轮用户输入。 |
| `reply` | string | 互动记录时是 | 本轮角色回复；当前仅保留接口，尚未用于打分。 |
| `flagged` | boolean | 互动记录时是 | 是否命中禁用词。 |
| `turn` | int | 互动记录时是 | 当前轮次。 |
| `relationship_type` `preferred_addressing` `boundaries_summary` | string | admin 编辑时是 | 人工更新的关系信息。 |
| `SafetyPolicy.prompt_notice()` | string | 聊天 prompt 时是 | 将关系类型映射为可执行表达边界。 |

### 输出
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `RelationshipState` | dataclass | 是 | 包含 intimacy、trust、familiarity、interaction_quality_recent、preferred_addressing、relationship_type、boundaries_summary、dependency_risk、updated_at、change_reasons。 |
| `summary_for_prompt()` | string | 是 | 注入人格层的关系摘要。 |
| `boundary_summary()` | string | 是 | 返回 Relationship 页展示的关系类型表达边界摘要。 |
| `data/runtime/<id>/relationship.json` | file | 否 | 关系状态持久化结果。 |
| `data/runtime/reminders/<character_id>.json` | file | 否 | reminder 仓库，保存 `id`、`title`、`note`、`due_at`、`repeat_rule`、`status`、`last_triggered_at` 等字段。 |
| `data/runtime/proactive/settings.json` | file | 否 | 主动陪伴设置持久化结果。 |
| `data/runtime/proactive/events.jsonl` | file | 否 | 主动行为、抑制理由和用户反馈日志。 |
| `PerceptionContext.dnd_reason` | string | 否 | 感知隐私层传入的主动勿扰原因。 |

## 流程
1. `ConversationService._prepare_turn()` 在构造 `PromptContext` 时调用 `summary_for_prompt(character_id)`。
2. prompt 中的关系块只提供上下文补充，不直接决定人格。
3. `ConversationService` 在关系块之后注入 `SafetyPolicy.prompt_notice()`，使关系类型变成可执行表达边界。
4. `ConversationService._finalize_turn()` 在回复完成后调用 `record_interaction()`。
5. `record_interaction()` 先根据用户输入和 `flagged` 计算 `interaction_quality_recent` 的平滑值。
6. 系统按保守规则缓慢增加 `familiarity`，并在满足条件时小幅调整 `trust`、`intimacy`、`dependency_risk`。
7. 每次更新都会在 `change_reasons` 中追加原因，并写入 `relationship.json`。
8. sidecar lifespan 启动时创建 `CompanionRuntime`，该 runtime 每轮先查询 `RoutineReminderService.due(now)`，再按可用性采集 `PerceptionContext`。
9. 已到期 reminder 被转换成 `scene=reminder`、`reason=reminder_due:<id>` 的 `ProactiveSignal`，并携带 `reminder_id` 与标题摘要 metadata。
10. scheduler 读取 `data/runtime/proactive/settings.json`、最近 `events.jsonl`、当前 `EmotionState.emotion_summary` 和 `PerceptionContext.dnd_reason`，决定是发送、降级还是抑制主动行为；collector 不可用时仍允许 reminder 信号继续经过策略层。
11. 允许发送时，runtime 通过 WebSocket 广播 `proactive` 帧；`full` 级别优先尝试 LLM 生成，两次链路分别可降级为模板短句、表情或沉默。reminder 的完整 `note` 不进入 proactive 日志。
12. 一次性 reminder 到期后进入 `pending_ack`；`daily` / `weekly` reminder 到期后立即推进下一次 `due_at`。
13. admin 可通过 `/admin/relationship/{character_id}` 管理关系状态，通过 `/admin/proactive/*` 管理主动陪伴设置、状态、日志和测试动作，通过 `/admin/reminders/*` 管理 reminder 的创建、改期、完成和删除。
14. `/admin/proactive/feedback` 命中 reminder 事件时，`知道了` 会完成 reminder，`稍后提醒` 会顺延 30 分钟，`今天别提醒` 只记录主动反馈不额外改 reminder 数据。

## 约束与规则
- 数值字段使用 0..100 范围钳制，不允许越界。
- 允许的 `relationship_type` 只有 `friend`、`partner`、`family`、`mentor`、`coworker`；非法值回退为 `friend`。
- `friend` 关系禁止恋爱占有、现实承诺和排他表达。
- `partner` 关系允许更亲密语气，但禁止现实监护承诺、嫉妒控制和隔离现实关系。
- `family` 关系禁止恋爱化表达。
- `mentor` 关系禁止依赖绑定和越权专业承诺。
- `coworker` 关系禁止强亲密、排他和占有表达。
- 关系更新规则刻意保守，避免一两轮对话造成剧烈变化。
- `reply` 参数当前保留给后续评分优化，现阶段不参与计算。
- 主动行为不写入普通工作记忆，不增加普通 `turn`；只有用户回应后，回应文本才进入常规聊天链路。
- 主动陪伴设置以本地文件为权威来源；runtime 每次循环重新读取设置，允许管理界面修改后在运行时生效。
- 主动行为只能读取当前情绪摘要影响表达方式和打扰等级，不能绕过策略层的冷却、勿扰和每日上限。
- 当 `PerceptionContext.dnd_reason` 非空时，主动策略必须返回 `suppressed_by=privacy_dnd:<reason>`，且不得广播 proactive 帧。
- reminder 与关系状态独立存储；它不写入当前记忆仓库，也不直接修改 `RelationshipState`。
- reminder proactive 日志只允许记录 `reminder_id`、标题摘要和原因；完整 `note` 只保存在 reminder 仓库。
- 当前没有承诺系统和事件记忆时间线。

## 验收标准
- 首次读取某角色关系状态；自动生成默认 `relationship.json`。
- 连续进行正向互动且轮次满足条件；`familiarity` 与部分 `trust` / `intimacy` 发生小幅增长。
- 本轮 `flagged=true`；`interaction_quality_recent` 降低，且亲密度不会继续增长。
- 用户输入出现依赖性表达；`dependency_risk` 小幅上升。
- 调用 admin 重置接口；关系状态回到默认值并记录重置原因。
- 调用 `boundary_summary("mentor")`；返回文本包含导师关系和禁止依赖绑定。
- 用户不发送消息但保持 sidecar 运行；满足主动条件后，主窗口收到 `proactive` 帧且 `events.jsonl` 增加一条 sent 记录。
- 创建一条一次性 reminder 并等待到期；该 reminder 进入 `pending_ack`，随后点击“知道了”后变为 `completed`。
- 创建一条 `daily` reminder 并等待到期；返回的 proactive 记录包含 `reason=reminder_due:<id>`，且 reminder 的下一次 `due_at` 自动推进到次日同一时刻。
- 在管理界面关闭主动陪伴或进入勿扰时间；runtime 记录 suppressed 决策，且主窗口不收到气泡。
- 配置感知隐私勿扰规则并满足主动条件；runtime 记录 `suppressed_by=privacy_dnd:<reason>`，且主窗口不收到气泡。
