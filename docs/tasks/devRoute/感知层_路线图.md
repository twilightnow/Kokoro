# 感知层路线图

← [返回总览](开发路线图.md)

设计参考：[感知层设计](../../desgin/感知层.md)

---

## 模块职责

- **负责**：采集环境上下文（窗口、时间、活跃状态）、运行触发器、输出结构化感知上下文
- **不负责**：决定角色说什么（由人格层决定）
- **约束**：感知层只向人格层输出结构化上下文描述（场景标签、时间段、活跃状态），不直接引用人格层的类型或状态机，保持层间解耦

---

## 分级感知策略

```
Level 0：进程级感知（本地，零成本）
├── 当前活动窗口标题
├── 用户活跃状态（键鼠事件）
└── 系统时间（深夜/工作时间判断）

Level 1：内容级感知（本地OCR或窗口API，低成本）
├── 当前应用类型判断（游戏/代码/浏览器）
└── 简单文字内容提取

Level 2：视觉理解（Gemini Flash API，按需调用）
└── 复杂场景理解，仅在 L0/L1 无法判断时触发
```

首版只实现 Level 0，Level 1 按需补充，Level 2 后置。

## Phase 0–1：感知层暂不启动

在 Phase 0（收口 Demo）和 Phase 1（人格层定型）期间，感知层维持占位状态，不接入任何真实感知能力。

主要原因：

- 感知触发的主动介入需要稳定的人格层支撑
- 此阶段资源应聚焦人格稳定性验证

Phase 0–1 的唯一任务：

- [ ] 定义 `PerceptionContext` 数据结构（空实现即可），确保人格层可以接收但忽略
- [ ] `prompt_builder.py` 预留 `perception_context` 注入点（可选，为空时不影响 prompt）

## Phase 3：Level 0 感知接入

### 3.1 感知上下文对象

目标：定义统一的结构化感知输出，供人格层消费。

```python
@dataclass
class PerceptionContext:
    window_title: str           # 当前活动窗口标题
    app_category: str           # "coding" / "browser" / "game" / "unknown"
    time_of_day: str            # "morning" / "afternoon" / "evening" / "late_night"
    user_active: bool           # 最近 N 分钟内有键鼠事件
    idle_minutes: int           # 用户空闲时长（分钟）
    scene_tags: list[str]       # 高层场景标签，如 ["late_night", "long_work"]
```

任务：

- [ ] 在 `src/perception/context.py` 中定义 `PerceptionContext` dataclass
- [ ] 实现 `scene_tags` 的推断逻辑（基于其他字段生成标签列表）

### 3.2 Level 0 感知采集

依赖库：

- 窗口标题：`pygetwindow`（仅 Windows）
- 键鼠活跃：`pynput`

任务：

- [ ] 实现 `src/perception/collectors/window_collector.py`：`get_active_window_title() -> str`
- [ ] 实现 `src/perception/collectors/activity_collector.py`：后台监听键鼠事件，维护最后活跃时间戳，提供 `get_idle_minutes() -> int`
- [ ] 实现 `src/perception/collectors/time_collector.py`：`get_time_of_day() -> str`（基于当前小时推断）
- [ ] 实现 `src/perception/context_builder.py`：整合各 collector 输出，生成 `PerceptionContext`

### 3.3 触发器系统

目标：基于感知上下文触发主动介入事件，传递给人格层。

```python
class ProactiveTrigger:
    cooldown: int = 1800  # 默认冷却 30 分钟

triggers = [
    IdleTrigger(threshold_minutes=120),          # 空闲超 2 小时
    LateNightTrigger(hour_range=(23, 4)),         # 深夜工作（23:00–04:00）
    LongWorkTrigger(threshold_minutes=60),        # 连续工作 1 小时
    WindowSwitchTrigger(freq_per_min=10),         # 高频切窗（可能在纠结）
]
```

任务：

- [ ] 定义 `BaseTrigger` 抽象类：`check(context: PerceptionContext) -> bool`，`get_event_tag() -> str`
- [ ] 实现 `IdleTrigger`、`LateNightTrigger`、`LongWorkTrigger`（首版优先）
- [ ] 实现冷却机制：每个触发器独立记录上次触发时间，冷却期内 `check()` 直接返回 False
- [ ] 实现 `src/perception/trigger_manager.py`：轮询所有触发器，返回已触发的事件标签列表
- [ ] 触发和抑制事件均写入日志（区别于对话日志，写入 `logs/perception_*.jsonl`）

### 3.4 人格层接入感知上下文

任务：

- [ ] 在 `prompt_builder.py` 中实现 `perception_context` 注入：将场景标签和时间段以自然语言描述注入 prompt
- [ ] 人格层根据感知事件标签决定是否主动发言（逻辑在人格层，不在感知层）
- [ ] 主动介入仅生成一句话 + 快捷回应建议，不默认展开多轮对话

### 验收

- 空闲 2 小时后触发 `IdleTrigger`，冷却期内同一触发器不再触发
- `PerceptionContext` 包含准确的窗口标题、时间段、活跃状态
- 触发日志和抑制日志可从 `logs/` 中查阅
- 感知层关闭后，人格层和 CLI 对话流程不受影响

## 不建议在此阶段做的事

- Phase 3 之前接入任何真实感知能力（人格层未稳定前，感知触发的介入无意义）
- 高频实时评论用户行为
- 基于截图内容做细粒度判断（后置）
- 长时间持续监听并密集输出反馈
