# 感知层

所属项目：[[Kokoro]]

---

## 分级感知策略（核心成本控制手段）

```
Level 0：进程级感知（本地，零成本）
├── 当前活动窗口标题
├── 用户活跃状态（键鼠事件）
└── 系统时间（深夜/工作时间判断）

Level 1：内容级感知（本地OCR或窗口API，低成本）
├── 当前应用类型判断（游戏/代码/浏览器）
└── 简单文字内容提取

Level 2：视觉理解（Gemini Flash API，按需调用）
└── 复杂场景理解，仅在L0/L1无法判断时触发
```

首版只建议实现 Level 0，必要时补少量 Level 1。复杂视觉理解放后期，不进入第一阶段主线。

感知层的目标不是“尽量多看见”，而是为少数高价值场景提供足够可靠的上下文，让角色能在合适时机自然开口。

## 主动介入触发器

```python
class ProactiveTrigger:
    cooldown: int = 1800       # 冷却时间（秒），防止烦人

    triggers = [
        IdleTrigger(threshold=7200),        # 空闲超2小时
        LateNightTrigger(hour_range=(23,4)), # 深夜工作
        GameDetectedTrigger(),               # 后续扩展：优先用窗口标题关键词粗判
        WindowSwitchTrigger(freq=10),        # 频繁切换窗口（10次/分钟以上，可能在纠结）
        LongWorkTrigger(threshold=3600),     # 连续工作1小时
    ]
```

冷却机制确保主动介入不会太频繁，每次触发后进入冷却，防止明日香变成烦人的弹窗。

补充约束：

- 默认不主动打断用户
- 同一类场景在冷却期内只触发一次
- 只有高置信度、低打扰的场景才允许主动介入
- 感知层只向人格层输出**结构化上下文描述**（如场景标签、时间段、活跃状态），不直接引用人格层的类型或状态机，保持层间解耦

推荐首版重点覆盖的使用场景：

- 深夜仍在工作或持续活跃
- 长时间连续使用电脑
- 长时间空闲后重新回到电脑前
- 高频切换窗口，可能处于犹豫或分心状态

不建议首版追求的方向：

- 高频实时评论用户行为
- 基于截图内容做细粒度判断
- 长时间持续监听并密集输出反馈

首版建议仅支持以下输入：

- 当前活动窗口标题
- 用户活跃 / 空闲状态
- 系统时间段

其中“游戏检测”若在首版启用，也仅基于窗口标题关键词做粗略判断，不依赖截图或 OCR。

OCR、截图理解、直播陪看都作为后续扩展，不阻塞主线开发。
