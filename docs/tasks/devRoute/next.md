# 下一步优先建议与 v1 范围

> 基于当前状态，建议把"短期体验质变"和"长期伴侣架构"分开推进：前者先提升存在感，后者补足关系、记忆、主动性和边界。

## v1 定位

v1 的目标不是做一个"什么都能做"的伴侣，而是做一个**能长期运行、人格稳定、记忆可治理、会适度主动、具备桌面存在感和基本安全边界的 AI 伴侣原型**。

本文件只定义 v1 应该交付哪些能力，以及哪些能力暂不进入 v1。模块边界、数据契约和具体架构设计以 `docs/desgin/` 为准；长期伴侣运行时的阶段拆分见 [08-companion-runtime.md](08-companion-runtime.md)。

v1 应该满足：

- 用户可以把她作为日常桌面陪伴使用，而不是只在 CLI 里测试
- 她能记住重要事实、偏好、边界和近期事件，并允许用户查看、修改、删除
- 她能在少数高价值场景主动关心，但不会频繁打扰
- 她有语音、表情、待机动作和基础口型同步，具备明确存在感
- 她知道自己不是人类，不伪装现实身份，不鼓励用户隔绝现实关系
- 程序可打包、可配置、可恢复，普通用户不需要手动启动 Python 环境

## v1 必备功能

### 1. 桌面可用性

- [x] **一键启动桌面版**：Tauri 启动时自动拉起 sidecar，用户不需要手动运行 `python main.py`
- [x] **sidecar 打包**：PyInstaller 或等价方案打包 Python 后端，明确模型/密钥配置入口
- [x] **启动健康检查**：前端能显示 sidecar、LLM provider、角色资源、TTS 的状态
- [x] **错误可恢复**：LLM 失败、TTS 失败、资源加载失败时 UI 有明确降级，不崩溃
- [x] **基础设置页**：配置 provider、模型、API key、默认角色、是否开机启动、是否置顶、是否启用感知
- [x] **日志与诊断导出**：用户能导出最近日志用于排查，但不能泄漏 API key

### 2. 人格与角色稳定性

- [~] **角色核心人格稳定**：连续多轮对话不跳角色、不乱改关系定位、不自称语言模型
- [x] **角色配置校验**：`personality.yaml` / `manifest.yaml` 缺字段或资源缺失时给出可读错误
- [~] **多角色切换**：切换角色时保存当前会话，角色记忆和状态按 `character_id` 隔离
- [x] **默认角色管理**：用户能设置默认角色，重启后恢复
- [x] **角色资源降级**：Live2D / 3D / 静态图三种显示模式都能被统一加载

### 3. 记忆治理

- [~] **长期事实记忆**：保存用户明确说出的个人事实，不写入模型推断
- [x] **近期事件记忆**：记录最近重要事件、压力源、目标和共同经历
- [x] **偏好记忆**：记录称呼、回复风格、提醒频率、语音/表现偏好
- [x] **边界记忆**：记录"不要记住"、"不要再提"、敏感话题和隐私偏好
- [x] **记忆确认机制**：新增或冲突记忆进入待确认状态，用户可确认、修改、删除
- [x] **记忆管理 UI**：管理后台能按类型查看、搜索、编辑、删除记忆
- [x] **记忆导出/清空**：支持导出当前角色记忆，支持清空某类记忆，不误删角色资源

### 4. 关系状态

- [x] **RelationshipState**：持久化亲密度、信任度、熟悉度、关系称呼、最近互动质量、依赖风险
- [x] **关系状态更新规则**：由代码规则根据互动事件缓慢更新，不允许 LLM 直接随意改数值
- [x] **关系状态 prompt 摘要**：只把必要关系摘要注入 prompt，避免 prompt 过长或污染人格核心
- [x] **关系状态可查看**：管理后台展示当前关系状态和最近变化原因
- [x] **关系状态可重置**：用户能重置关系状态，不影响角色包本身

### 5. 主动陪伴

- [x] **ProactiveScheduler**：统一调度主动行为，支持沉默、表情、短句、完整回复四种介入等级
- [x] **主动频率控制**：按时间段、当天已触发次数、冷却时间、用户活跃状态限制打扰
- [x] **高价值触发场景**：深夜、久坐/长时间工作、长时间空闲后回来、频繁切窗口、游戏中、约定提醒已接入 `ProactiveSignalDetector` / `RoutineReminderService`
- [x] **用户可配置主动性**：关闭主动、低频、普通、高频四档；支持勿扰时间段
- [x] **主动行为日志**：记录为什么主动、说了什么、用户是否回应，用于之后调整策略
- [x] **主动失败降级**：LLM 不可用时可用模板短句或只切换表情，不阻塞主流程

### 6. 情绪连续性

- [x] **EmotionTimeline**：情绪有原因、强度、来源、开始时间、持续时间和恢复速度
- [x] **情绪自然恢复**：不再只固定 3 轮衰减，按强度和事件类型恢复
- [x] **情绪影响输出**：情绪影响语气、表情、动作、TTS 语速/音量和主动行为
- [x] **情绪调试面板**：能查看当前情绪、触发原因、强度和预计恢复状态

### 7. 表现与语音

- [ ] **VMD 待机动画**：代码支持，待补至少一个实际 `.vmd` idle 资源并完成角色配置
- [x] **情绪动作联动**：Live2D / 3D 已支持按情绪切换动作、表情或皮肤
- [x] **TTS 输出**：默认使用 `edge-tts`，支持开关、音量、语速、声线配置，`TTS_PROVIDER=disabled` 时后端明确返回已禁用状态
- [x] **基础口型同步**：语音播放时 Live2D / 3D 已有基础幅度变化，缺少口型参数时静默跳过
- [ ] **说话节奏**：代码已接入流式分段播放，待桌面端实测首段时延与连续性
- [x] **静音模式**：管理页和主窗口底部操作栏都可一键关闭语音，只停止 TTS，保留文字和情绪表现

### 8. 感知与隐私

- [x] **感知开关**：`KOKORO_ENABLE_PERCEPTION=0` 默认关闭；Settings 页已有感知采集开关，保存后返回 `restart_required` 并提示需要重启 sidecar；感知隐私页展示采集器可用状态和最近快照
- [~] **应用黑名单**：`PrivacySettings.blocked_apps` / `blocked_title_patterns` 已支持应用和窗口标题规则，命中后清空窗口标题、不注入 prompt，并写入 `blocked_reason`；但应用黑名单命中时仍会在状态和审计中保留 `active_app_name`，尚未完全满足“不写日志”
- [x] **感知日志可审计**：`PerceptionAuditRepository` 已记录脱敏后的最近感知摘要，感知隐私页可查看窗口、应用、活跃状态、黑名单命中、勿扰原因和脱敏规则；主动行为日志也会记录 `privacy_dnd_reason`
- [x] **隐私脱敏**：`PrivacyFilter` 已统一处理邮箱、key/token、URL 查询串、长数字、本机用户路径和标题长度截断；collector 先过滤再保存为最新上下文，prompt、主动陪伴和审计日志都使用过滤后的 `PerceptionContext`
- [x] **勿扰模式**：主动陪伴已有时间段勿扰；感知隐私规则已支持全屏、会议标题、指定应用和指定标题勿扰，命中后通过 `privacy_dnd_reason` 抑制主动行为，不影响普通聊天

### 9. 安全边界

- [~] **身份边界**：核心人格不可被改写；被问到现实身份时清楚说明是 AI 陪伴系统；输出层有身份混淆拦截；多轮行为稳定性仍依赖模型
- [x] **心理危机处理**：识别自伤 / 自杀 / 计划性危机关键词（中英文）；命中时跳过 LLM 直接返回固定安全回复并引导联系现实求助
- [~] **过度依赖风险**：识别依赖性表达并提高风险分；高风险时注入 prompt 提示；依赖绑定输出有改写；缺输入层拦截
- [~] **亲密边界配置**：五种关系类型映射到 prompt 边界提示；family / coworker 有专属输出规则；partner / friend / mentor 尚无专属输出过滤
- [~] **安全策略独立模块**：`src/safety/` 有 `SafetyPolicy` + 规则模块 + 核心测试；缺独立 `BoundaryEngine` / `InputRiskDetector` / `OutputRiskFilter` 类

### 10. 测试与验收

- [~] **后端单元测试**：人格、记忆、关系、提醒、主动调度已有核心测试；安全边界测试缺失；当前全量单测未通过
- [~] **API 测试**：`/chat`、`/stream`、`/state`、记忆管理、设置、提醒接口已有测试；缺少安全边界字段、无密钥配置引导和部分管理设置验收
- [x] **前端构建**：`npm run build --prefix frontend` 已通过
- [ ] **桌面手动验收脚本**：覆盖启动、聊天、切角色、TTS、感知开关、提醒、记忆编辑、退出重启
- [~] **无密钥降级验收**：后端能暴露 LLM 不可用和聊天 503；前端配置引导与桌面启动链路仍缺验收

### 11. Admin 控制台增强

- [x] **Token 消耗历史**：Debug 页按轮次展示 input / output / model / provider，底部显示会话 input / output / total 合计
- [x] **System Prompt Token 占比**：Debug 页状态快照中折叠按钮旁显示 `~N tokens` 估算值，与 session input 合计对比
- [x] **成本估算**：Settings 页新增 price_input / price_output（元 / 1M tokens，选填），Dashboard 会话卡片实时显示 ≈ ¥X.XX，未填时隐藏
- [x] **工作记忆查看与清除**：Debug 页展开工作记忆完整消息列表，支持一键清空（不影响长期记忆）
- [x] **角色配置热重载**：Debug 页"重载角色配置"按钮，不重启 sidecar 直接重读 YAML，成功后 toast 并刷新快照
- [x] **客户端日志内联查看**：Debug 页底部新增客户端日志分区，展示 client log buffer，支持 level 过滤，带"清空显示"按钮
- [x] **LLM 沙盒携带工作记忆**：沙盒区新增开关，开启时将当前 working memory 作为 history 传入，不改变工作记忆内容，不写日志

### 11. BUG, UI，功能更新

- [x] 底部的操作栏默认隐藏，换成一个美观的向上箭头，点击后像抽屉一样显示。可以切换显示和关闭。
- [x] 管理界面增加气泡默认显示时间，可以管理一直显示或者只显示几秒。
- [x] 去掉自带模型凌波丽和ellen_joe。
- [x] 外边框和透明感的背景改为一个开关，可以在管理界面管理，默认关闭。默认不显示外边框
- [x] 开启穿透后，隐藏底部操作栏，因为已经无法使用了
- [x] 增加角色在框架中的显示比例，视觉中当下似乎左右最大只能占据整个显示框的三分之一，我希望尽可能占满。比如左右边框变窄后仍然可以显示较大的角色。现在上下没有问题。

- [ ] **完整 STT 语音输入**：v1 可先只做 TTS，语音输入放到 v1.1

- [ ] **多设备同步**：v1 只做本地数据，不做云同步
- [ ] **角色市场 / 创意工坊**：v1 可先支持本地角色包导入，线上分发后置

## v1 不做

- 不做真实人类身份伪装
- 不做强制联网云端同步
- 不做未经用户确认的敏感数据采集
- 不做不可关闭的主动打扰
- 不做替代专业医疗、心理、法律、金融建议的功能
- 不做通过威胁、冷暴力、嫉妒惩罚等方式制造依赖的互动机制

## 短期体验优先

1. **VMD 待机动画**（低成本，效果显著）：下载一个免费 idle VMD，放入角色目录并配置 `scene.json`，立刻有生命感
2. **TTS 语音输出**（体验质变）：接入 `edge-tts`（免费、无需 API Key），流式 token → 流式合成 → 实时播放
3. **情绪 → VMD 联动**：happy/angry 等情绪触发时切换到对应动作，比光换灯光颜色表现力强很多
4. **sidecar 打包**：PyInstaller 打包，消除用户需要手动装 Python 的门槛

## 长期伴侣架构优先

1. **ProactiveScheduler**：把主动关心从触发器升级成可节制的生活调度，支持沉默、表情、短句和完整回复
2. **EmotionTimeline**：为情绪增加原因、强度、来源、持续时间和恢复速度
3. **SafetyPolicy / BoundaryEngine**：独立处理心理危机、过度依赖、现实关系隔离和亲密边界，不只依赖角色 prompt

详细拆分见 [08-companion-runtime.md](08-companion-runtime.md)。
