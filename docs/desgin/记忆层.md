# 记忆层

所属项目：[[Kokoro]]

---

## 技术选型

| 组件 | 方案 | 许可证 |
|------|------|--------|
| 向量数据库 | ChromaDB | Apache 2.0 |
| 记忆管理 | mem0 | Apache 2.0 |
| 关系状态存储 | SQLite | Public Domain |

首版目标不是“记得很多”，而是“记得准，不乱记”。

---

## 三级记忆结构

```
长期记忆
├── 用户基本信息（名字、职业、习惯）
└── 关键关系节点（第一次对话、重要事件）

摘要记忆
├── 历史对话摘要
└── 承诺/约定记录

工作记忆
└── 最近10轮对话
```

---

## 关键设计

**记忆按角色隔离：** 每个角色独立的ChromaDB collection，互不干扰。

**写入原则：**

- 用户明确说出的事实，才进入长期记忆
- 模型推断出的内容，默认只进入摘要记忆
- 情绪性、一次性的内容，不默认进入长期记忆
- 发生冲突时保留旧记录并标记待确认，不直接覆盖

**实现建议：**

- 首版以工作记忆 + 会话摘要 + 少量结构化长期记忆为主
- `mem0` 之类自动提取能力作为后续增强，不作为首版依赖
