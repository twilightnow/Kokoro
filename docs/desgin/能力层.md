# 能力层

能力层负责封装外部或本地能力，例如 LLM、TTS、STT、Embedding 和视觉理解。能力层只提供能力，不决定业务流程。

## 职责

能力层负责：

- provider 适配
- 鉴权和配置读取
- 统一请求/响应结构
- 错误归一化
- 流式能力封装
- 本地或云端能力的降级策略

能力层不负责：

- 决定什么时候调用 LLM
- 生成角色 prompt
- 写入记忆
- 修改关系状态
- 判断主动行为
- 直接操作 UI

## LLM 能力

LLM 是当前核心能力。

统一抽象应包含：

- `chat(system_prompt, messages)`
- `stream_chat(system_prompt, messages)`
- provider 名称
- model 名称
- token usage
- finish reason
- 错误类型

设计原则：

- provider 差异收敛在能力层内部
- 上层不关心 base_url、headers 和 SDK 差异
- CLI provider 和 API provider 返回结构一致
- 真实网络调用不进入单元测试

## TTS 能力

TTS 负责把角色文本转成语音。

统一抽象建议：

- 输入文本
- voice
- rate
- volume
- format
- 返回音频流或音频二进制

设计原则：

- TTS 失败不影响文字回复
- 支持用户静音
- 支持角色默认声线
- 支持按情绪调整语速和音量
- 不把 TTS 逻辑写进人格层

## STT 能力

STT 负责语音输入。它不是 v1 的必要核心，但设计上应预留。

设计原则：

- 麦克风必须显式开启
- 不做默认常驻监听
- 语音识别结果进入普通用户输入链路
- 原始音频是否保存必须由用户配置

## Embedding / 检索能力

Embedding 可以用于长期记忆检索，但不应成为早期记忆系统的前置依赖。

设计原则：

- 结构化记忆优先
- 向量检索用于补充召回，不替代记忆治理
- embedding provider 应与 LLM provider 解耦
- 检索结果必须经过 token budget 和敏感过滤

## 视觉理解能力

视觉理解属于高敏感能力。

设计原则：

- 不默认开启
- 不默认截图
- 必须有显式权限和可见提示
- 截图、OCR、视觉模型结果都应经过隐私过滤
- 只输出场景摘要，不直接写入长期记忆

## 错误与降级

能力调用失败时应归一化错误：

- 配置缺失
- 鉴权失败
- 网络失败
- provider 限流
- 内容被拒绝
- 模型不可用
- 本地依赖缺失

降级原则：

- LLM 失败：返回可解释错误，不破坏会话状态
- TTS 失败：继续显示文字
- STT 失败：保留文本输入
- 视觉失败：不影响对话
- Embedding 失败：退回结构化记忆

## 配置边界

能力层读取 provider 配置，但不应把密钥暴露给前端。

设计要求：

- API key 只在后端处理
- 管理界面读取配置时不回显真实 key
- provider reload 只影响能力层实例
- 角色包不包含真实密钥

## 设计约束

- 能力层是适配层，不是业务层
- 新能力优先新增独立 client，不堆进单个文件
- 上层只依赖稳定抽象
- 任何能力都必须有失败降级路径
- 高敏感能力必须有显式权限、日志和关闭开关
