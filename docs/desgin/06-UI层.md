# UI 层

## 目标
定义 Kokoro 当前前端 UI 的真实职责: 主窗口负责对话、角色呈现和主动气泡，管理界面负责观测、配置和调试。

## 范围
### 包含
- `frontend/src/App.vue` 主窗口的状态同步、输入、角色显示和 Tauri 交互。
- `frontend/src/components/` 中 `SpritePanel`、`BubbleBox`、`InputBar` 的当前分工。
- `frontend/src/admin/` 管理界面的 hash 路由和视图边界。

### 不包含
- 不定义人格、记忆、关系或 LLM 规则。
- 不把尚未实现的独立移动端、Web 站点或多窗口复杂布局写成现状。

## 输入 / 输出
### 输入
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `/state` 响应 | `StateResponse` | 是 | 主窗口初始化和轮询状态源。 |
| `/chat` `/stream` 响应 | JSON / WebSocket | 是 | 主窗口对话回复、token 流和可选 `emotion` 摘要。 |
| `/stream` `proactive` 帧 | WebSocket | 否 | 后台主动陪伴广播，包含 level、scene、content、expression、actions 和可选 `emotion` 摘要。 |
| `/health` 和 `/admin/*` 响应 | JSON | 管理界面时是 | dashboard、settings、debug 等页面的数据来源。 |
| `/admin/perception/*` 响应 | JSON | 管理界面时是 | 感知隐私设置、状态和脱敏审计的数据来源。 |
| `/admin/reminders/*` 响应 | JSON | 管理界面时是 | reminder 管理页的数据来源。 |
| Tauri 事件 | event | 否 | 主窗口 focus、角色切换、关闭请求、admin 窗口操作。 |
| `localStorage` | key-value | 否 | 保存穿透、置顶、窗口缩放、角色缩放、角色偏移、TTS 开关、输入栏可见状态、外边框开关、气泡时长、上次角色等本地偏好。 |
| `display` 配置 | object | 是 | `placeholder`、`image`、`live2d`、`model3d` 的展示配置。 |

### 输出
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 渲染结果 | DOM / Canvas / WebGL | 是 | 主窗口显示角色、气泡和输入栏。 |
| sidecar 请求 | HTTP / WebSocket | 是 | 发送聊天、同步状态、管理配置、导出诊断。 |
| Tauri 命令 | invoke | 否 | 设置穿透、置顶、开机启动、窗口位置和尺寸。 |
| `localStorage` 更新 | key-value | 否 | 记录本地交互偏好。 |

## 流程
1. 主窗口 `App.vue` 启动时读取本地偏好，先检测是否存在 Tauri 运行时。
2. 若存在 Tauri 运行时，主窗口再动态导入 `@tauri-apps/api/*` 并应用穿透、置顶和窗口尺寸设置。
3. 启动时立即从 `kokoro-last-character` 恢复上次使用的角色信息（id、name、display、turn），消除首帧闪白。
4. 主窗口调用 `init()` 与 `syncState()` 建立聊天状态，并周期性刷新 `/state`；完成后更新并持久化最新角色信息。
5. 主窗口收到 display 配置后，由 `SpritePanel` 按 `display.mode` 选择 `Placeholder`、`Image`、`Live2DCanvas` 或 `Model3DCanvas`。
6. 底部输入区在穿透模式（`kokoro-passthrough-lock=1`）时完全隐藏；非穿透模式下仅显示向上箭头切换按钮，点击展开/收起 `InputBar`，展开状态由 `kokoro-input-bar-visible` 持久化，默认折叠。
7. `InputBar` 提交文本后通过聊天 composable 发送请求，`BubbleBox` 展示最新回复；同一区域也负责主窗口语音输入和 TTS 一键静音入口。
8. 气泡显示时长由 `kokoro-bubble-duration`（秒，0=常驻）控制；时长 >0 且非主动陪伴消息时，回复完成后自动清空气泡。
9. 主窗口在 `thinking` / `done` / `proactive` 帧中消费 `emotion` 摘要，用于同步当前情绪并为 TTS 计算 `rate`、`volume`。
10. 主窗口收到 `proactive` 帧后，按 `level` 处理: `expression` 只更新表情；`short` 显示短气泡并自动淡出；`full` 显示可保留的主动短句并允许 TTS。
11. 主窗口点击主动气泡快捷回应时，先调用主动反馈接口记录 `event_id` 和 `feedback`，再把用户点击文本作为普通聊天消息发送。
12. 当托盘触发角色切换事件时，主窗口请求 `/switch-character`，然后重置 store 为新角色。
13. 管理界面 `frontend/src/admin/router.ts` 使用 hash 路由，按页面调用 `/state`、`/health` 和 `/admin/*`；其中 `Proactive` 视图独立管理主动陪伴策略，`Perception` 视图独立管理感知隐私规则和脱敏审计，`Reminders` 视图独立管理 reminder 内容。
14. `Dashboard` 轮询展示状态总览；`Settings` 和 `Interaction` 修改配置与本地偏好；其余视图处理角色、记忆、关系、主动陪伴、reminder、日志、统计和调试。

## 约束与规则
- UI 不得自行拼接角色资源本地路径，只能使用 sidecar 返回的 URL。
- 浏览器模式必须可运行；所有 Tauri API 调用前必须检查 `window.__TAURI_INTERNALS__?.invoke`。
- 主窗口只负责表现和交互，不负责人格推理、记忆存储或关系更新。
- 管理界面与主窗口共享部分 `localStorage` 键，但不共享服务实例。
- 角色缩放、偏移、窗口缩放属于本地偏好，不写回角色包或 `.env`。
- 外边框与透明背景效果由 `kokoro-show-frame` 控制，默认关闭（0）；启用时主窗口 `.app` 加 `.app--framed` 类，添加淡色边框和磨砂背景。
- 气泡自动隐藏计时器仅对 `proactiveLevel === 'silent'` 的普通回复气泡生效；主动陪伴短句（`short`/`expression`）由 `useChat.ts` 内部计时器独立管理。
- `Interaction` 视图只修改窗口级偏好和系统开机项；主动陪伴策略统一由 `Proactive` 视图通过 `/admin/proactive/*` 管理，感知隐私规则统一由 `Perception` 视图通过 `/admin/perception/*` 管理，reminder 内容统一由 `Reminders` 视图通过 `/admin/reminders/*` 管理。
- 主窗口 TTS 请求必须由调用方显式传入情绪导出的 `rate`、`volume`；前端不得依赖后端隐式读取全局情绪状态。
- 主窗口静音只关闭 TTS 播放与排队中的语音，不清除当前文字气泡、情绪状态或动作；口型同步应在停止播放后归零。

## 验收标准
- 在普通浏览器打开主窗口或管理界面；页面可加载且不会因缺少 Tauri API 抛出运行时错误。
- 主窗口收到 `/state.display.mode=model3d`；`SpritePanel` 切换到 `Model3DCanvas`，而不是继续显示占位图。
- 主窗口收到 `type=proactive, level=expression` 的帧；角色表情切换且不显示残留文本气泡。
- 修改 `localStorage` 中的 `kokoro-main-always-on-top`；主窗口响应 storage 事件并尝试同步窗口状态。
- 点击主窗口底部语音按钮切换到静音；后续回复继续显示文字，且不再发起新的 `/tts` 播放请求。
- 通过管理界面切换角色；主窗口收到角色切换事件后重置当前角色和 display 配置。
- 在 `Proactive` 页面保存设置并发送测试动作；状态卡片和最近日志更新，主窗口可收到测试主动气泡。
- 在 `Perception` 页面保存标题黑名单；刷新页面后规则仍显示，审计列表只显示脱敏后的窗口标题。
- 在 `Reminders` 页面创建一条一次性 reminder；列表出现新条目，等待到期后主窗口可收到 reminder proactive 气泡。
